## 2. 需求详情

### 2.1 学员注册入驻（新增）

**页面**：`Register.html`（新建）

**入口**：
* Home.html 首页顶部导航「注册入驻」
* 首页 hero 区「注册入驻」按钮

**业务规则**：

注册时选择身份类型，不同身份字段要求不同：

| 字段 | Key | 学员 | 普通用户 | 说明 |
|------|-----|------|----------|------|
| 身份类型 | `userType` | ✅ 必选 | ✅ 必选 | `student` / `normal` |
| 昵称 | `nickname` | ✅ 必填 | ✅ 必填 | 展示名称 |
| 手机号 | `phone` | ✅ 必填 | ✅ 必填 | 同时作为登录账号 |
| 性别 | `gender` | ✅ 必填 | ✅ 必填 | 男/女 |
| 密码 | `password` | ✅ 必填 | ✅ 必填 | ≥6位，需二次确认 |
| 班级 | `className` | ✅ **必填** | ✗ 不显示 | 剪辑一班/二班/调色班/特效班/综合班 |
| 接单角色意向 | `takerRole` | ✅ 必填 | ✅ 必填 | 剪辑师/调色师/特效师 |
| 自我介绍 | `introduction` | ○ 选填 | ○ 选填 | 审核参考 |
| 作品集附件 | `portfolio` | ○ 选填 | ✅ **必传** | 普通用户须上传作品集 |
| 同意协议 | `agreeTerm` | ✅ 必勾选 | ✅ 必勾选 | 平台入驻协议 |

**状态流转**：

```
填写表单 → 提交 → 写入 pendingStudents（status: Pending）
                      ↓
                运营在后台审核
              ↙            ↘
       通过 → talentPool      驳回 → 记录原因
      （status: Training）    （status: Rejected）
```

**校验规则**：
* 手机号查重：同一手机号不可重复提交
* 普通用户未上传作品集时提交按钮禁用
* 密码需二次确认一致

### 2.3 项目阶段进度条（新增）

**页面**：`ProjectSpace.html`（交付与修改 Tab）

**UI 说明**：

在交付 Tab 顶部新增横向阶段进度条：

```
精剪中 → 内部一审 → 内部二审 → 导演审 → 监制审 → 平台审 → 审核完成
```

**交互规则**：
* 绿色圆点 = 已完成阶段；蓝色圆点 = 当前阶段；灰色圆点 = 未开始
* **默认选中当前项目阶段**，下方仅展示该阶段的交付版本
* 点击其他阶段节点切换查看该阶段的版本
* 已选中的阶段再次点击不会取消（始终有一个阶段被选中，无「全部」状态）
* 阶段筛选与集数筛选联动（双重筛选）

**版本卡片增强**：
* 新增蓝色阶段标签（如「内部一审」）
* 继承版本显示灰色「🔗 继承」标签

### 2.4 阶段切换与版本继承（新增）

**操作入口**：`ProjectManagement.html` → 项目列表「项目状态(二级)」下拉框

**权限**：仅管理员/运营可操作（后台项目管理页）

**阶段切换时系统动作**：

1. 更新项目的 `projectStage` 字段
2. 同步更新项目协同空间的 `projectStage`
3. 记录 `stageHistory`：`{ from, to, time, operator }`
4. **自动生成继承版本**：

```
遍历旧阶段下每集的最新版本
  ├── 新阶段下该集已有版本 → 跳过
  └── 新阶段下该集无版本 → 生成继承记录
```

**继承版本数据结构**（delivery 新增字段）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `stage` | string | 该版本所属的项目阶段 |
| `inherited` | boolean | 是否为继承版本（true=未修改，直接继承） |
| `inheritedFrom` | string | 继承自哪个 delivery.id |

**继承版本特征**：
* `uploader`: '系统继承'
* `changeNote`: '阶段切换继承自「{旧阶段}」'
* `status`: 继承原版本状态（Approved 保持，其他设为 Submitted）
* `inherited`: true

**兼容处理**：
* 旧 deliveries 无 `stage` 字段时，自动补全为当前 `projectStage`
* 旧 deliveries 无 `inherited` 字段时，默认 `false`

### 2.6 财务扣款改造（增强）

**页面**：`FinanceManagement.html`

**变更点**：

| 项目 | 变更前 | 变更后 |
|------|--------|--------|
| 扣款粒度 | 项目级（总扣款平均分配到订单） | **单笔订单级**（逐笔录入） |
| 扣款弹窗 | 单个金额输入 + 原因 | 表格展示项目下所有订单，每笔单独填写扣款金额 |
| 扣款校验 | 无 | 单笔扣款不得超过该订单金额 |
| 扣款原因 | Form 表单字段 | 底部统一 TextArea，必填 |
| 订单详情 | 无扣款列 | 新增「扣款」列，展示每笔订单扣款金额 |

**扣款弹窗字段**：

| 列 | 说明 |
|----|------|
| 订单号 | 订单唯一标识 |
| 接单人 | 学员姓名 |
| 订单金额 | 该订单应付金额 |
| 扣款金额 | 可编辑，min=0，max=订单金额 |

**影响范围**：扣款保存后同步影响学员收益、榜单数据。

### 2.7 数据看板（新增）

**页面**：`DataBoard.html`（新建）

**入口**：全局侧栏「📈 数据看板」

**包含模块**：

#### 2.7.1 整体概览（KPI 卡片）

| 指标 | 口径 |
|------|------|
| 短剧总数量 | 项目总数（adminProjects + mockProjects 去重） |
| 短剧总制作集数 | 所有订单 episodeCount 之和 |
| 待进集数 | 总制作集数 − 已交付集数 |

#### 2.7.2 短剧整体情况（饼图）

按「项目状况 + 城市」分组展示项目数量分布（使用 AntV G2）。

#### 2.7.3 短剧进度集数分布（柱状图）

展示待进 / 已交付 / 在制中三个状态下的集数对比。

#### 2.7.4 短剧项目情况（看板）

横向看板，每列对应一个「项目状况 + 城市」组合（如「在制中 (长沙)」），列内为项目卡片。点击跳转项目详情。

数据补全：当某状况+城市组合下无项目时，自动注入示例项目以保证看板样式完整。

#### 2.7.5 项目完成进度（看板）

按制作阶段分列：精剪中 / 内部一审 / 内部二审 / 导演审 / 监制审 / 平台审 / 审核完成。仅展示有项目的阶段。

#### 2.7.6 短剧具体完成进度（卡片网格）

每个项目一张进度卡片：项目名（链接）、阶段+城市、进度条（已交付/总集数）。

#### 2.7.7 短剧进审时间（时间线）

* 表格：订单号、项目、接单人、接单时间、截止时间、状态
* 甘特图：有订单数据时展示接单→截止时间条（AntV G2 interval + transpose）

## 4. 验收标准

### 4.1 学员注册
