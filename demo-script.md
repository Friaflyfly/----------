## 2. 场景 1：登录后看到什么 ⭐ 立"多空间"招牌（2 分钟）

**页面**：`platform-home-demander.html`

### 演示动作

1. 打开页面，让业务方扫一眼整体布局
2. **指顶部空间切换器**：
   > "这是我登录进来——不是直接进首页，而是先看到我**当前所处的工作空间**。我现在是'骋风天合 · 短剧制作中心'。"
3. **点开空间切换器**：
   > "昭岚在骋风天合下面有 4 个不同的身份场景：短剧制作中心、市场宣传组、IP 孵化组、还有企业管理员视角；同时她还有自己的个人空间。"
4. **切到"市场宣传组"**：
   - 注意顶部钱包从 ¥185,000 变成 ¥62,000
   - 欢迎区"绑定账户""扣款主体"全变了
   > "切空间 = 切**操作上下文**。同一个账号，在不同空间下看到的菜单、数据、能花的钱、要扣的账户都不一样。"
5. **切到"IP 孵化组"**：
   - 顶部出现 toast："该空间未配置预算，下单将回退至企业主账户"
   > "这就是平台规则的一种：团队没预算就**自动回退到企业主账户**。后面我会演示这个回退是怎么记账的。"
6. **切回"短剧制作中心"**

### 业务方可能会问

- Q: "这跟现在天合接单平台有啥区别？"
  - A: "天合接单平台只有一个'我'，所有钱都从一个钱包出。新平台的'我'有 5 个上下文，每个上下文绑不同的钱袋。"
- Q: "这是同一个登录账号吗？"
  - A: "是。一个手机号 / 一套密码，登录后通过空间切换器切上下文，无需多次登录。"

## 3. 场景 2：发布需求 ⭐ 立"多类目"招牌（3 分钟）

**页面**：从 P1 → 点 **"📤 发布需求"卡片** → `demand-publish.html`

### 演示动作

1. **指顶部蓝色提示条**：
   > "页面顶部已经写明：本次需求归属'短剧制作中心'，下单后扣款主体是'本团队预算账户'。这个写明很重要——避免用户提交完才发现钱从'错的账户'扣了。"
2. **指一级类目左栏**：
   > "新平台不再只接短剧。一级类目 5 个：视频、文案、图片、内容、资质——这是我们和业务方梳理出来的最大公约数。每一类下面有 3-5 个二级类目。"
3. **点'视频制作' → 切到'品牌宣传片'**：
   - 下方动态字段从"集数 / 单集时长 / 解说风格"换成"品牌名 / 片长 / 是否需要拍摄 / 投放渠道"
   > "**重点：不同二级类目对应不同的字段。**这不是 UI 的差别，是后端表单 schema 的差别。后续做撮合算法时，每个二级类目可以有自己的字段、自己的承制方匹配规则、自己的报价模型。"
4. **再切回'短剧解说'**，**演示标签**：
   - 修仙 / 古风 / 抖音 已默认勾上
   > "标签和类目分开。类目决定'卖什么'，标签决定'什么调性'。同一个'短剧解说'类目，标'修仙 + 古风 + 抖音'和标'都市 + 现代 + 视频号'，匹配出的承制方会完全不同。"
5. **服务等级 + 制作方式**：
   > "进阶级 × 特效制作。等级影响报价区间——你看下方的推荐报价 ¥10,000 - ¥30,000 是根据这两个变量推出来的。"
6. **快速填表**：
   - 项目名：保留默认或改一下
   - 需求描述：粘一段（≥30 字）
   - 预算：18000
   - 交付日期：随便选个 5 月初
7. **点"提交审核"** → 跳到 `demand-list.html`，新需求高亮闪烁

### 业务方可能会问

- Q: "公投机制呢？"
  - A: "公投机制 PDF 里有，但首期建议先做'指定 + 平台推荐 + 报价对比'三种。公投留在二期。"
- Q: "字段这么多业务方填不完怎么办？"
  - A: "客服/平台运营可以介入'核定'环节——这是 PDF 里第三层的需求审核与核定模块。"

## 4. 场景 3：报价对比 + 下单 ⭐⭐⭐ 杀手锏（5 分钟）

**页面**：从 P3 列表 → 找一条状态是"💬 待选承制方"的 → 点"对比报价"按钮 → `demand-quote-comparison.html`

### 演示动作

#### A. 报价对比

1. **指顶部摘要 + 推荐说明**：
   > "需求审核通过后，平台从 238 家承制方中匹配出 4 家邀请报价。这 4 家是基于类目 + 标签算出来的。"
2. **过 4 张报价卡**（30 秒/张）：
   - 星辰创作工作室：⭐ 平台推荐，¥18,000，14 天，影视级
   - 银河剪辑社：💰 最低价 ¥12,500，10 天，AI 高效
   - 锦鲤创意：¥28,000，21 天，影视级电影感
   - 翼程文化：¥16,000，⚡ 最快 12 天
   > "每家承制方的报价、交期、方案要点、合作历史都摆在一起。下方有详细对比表，方便多人决策。"
3. **指"详细对比"表**：
   > "实际场景下，这页是甲方多人 review 的地方——制作主管看方案、财务看价格、统筹看交期。"

#### B. 下单确认（最重要的部分） ⭐⭐⭐

4. **点"星辰创作工作室"卡片的"选定该方"** → 卡片变蓝
5. **点底部"下一步：下单确认"** → 弹出**下单确认 modal**
6. **重点指 modal 中间的扣款主体 badge**：
   > "这个 badge 是 PDF 4.2 节强调的核心。它告诉用户：**当前空间下下单，钱从哪个账户扣**。
   >
   > 现在我在'短剧制作中心'空间，扣款账户是'短剧制作中心 - 团队预算账户'，余额 ¥185,000，本次锁定 ¥18,000，锁定后剩 ¥167,000。"
7. **不关 modal，去顶部空间切换器，切到"市场宣传组"**：
   - badge 颜色从蓝变橙（实际还是蓝色 - team-budget），账户名变成"市场宣传组 - 团队预算账户"，余额变 ¥62,000
   > "看，**我没改任何输入，只是切了空间，扣款主体就变了**。这就是'扣款主体识别'。"
8. **再切到"个人空间"**：
   - badge 变成"个人账户" + 余额 ¥3,200
   - **本次锁定 ¥18,000 余额不足！** 提示"❌ 余额不足，无法完成锁定，请先充值或申请预算"
   - 确认按钮被禁用
   > "余额不够，平台直接拦下来——不能下单。"
9. **再切到"IP 孵化组"**（团队未配预算）：
   - badge 变成黄色，弹出橙色提示："当前空间未配置团队预算账户，回退至企业主账户结算"
   > "团队没预算的场景，平台**自动回退到企业主账户**结算，但订单创建时仍然记录'由 IP 孵化组发起'，方便审计。"
10. **切回"短剧制作中心"**，**点"确认下单（锁定预算）"**：
    - Toast："✅ 下单成功！订单 OD-xxx，¥18,000 已从短剧制作中心 - 团队预算账户锁定"
    - 2 秒后自动跳到钱包页

### 业务方可能会问

- Q: "锁定 ≠ 扣款？"
  - A: "对。锁定是'冻结'，钱还在团队预算账户里只是不能再花；订单验收通过后才正式从锁定金额走结算分账。"
- Q: "如果下单后撤回呢？"
  - A: "锁定释放回原账户。如果是部分退款，那就部分释放、部分入账给承制方。"

## 5. 场景 4：钱包总览 + 团队预算划拨（4 分钟）

**页面**：自动跳到 `wallet-overview.html`（也可从顶部钱包 chip 点进）

### 演示动作

1. **指最上方"当前空间绑定账户"大卡**：
   > "钱包页第一眼看到的不是'我所有的钱'，而是**当前工作空间正在用哪个账户**。换句话说，钱包是按上下文聚焦的，不是一个大杂烩。"
2. **看下方"全部资金账户"矩阵**：
   - 企业主账户 ¥1,250,000
   - 短剧制作中心 - 预算 ¥185,000（已锁 ¥28,000 + 18,000 = ¥46,000）
   - 市场宣传组 - 预算 ¥62,000（已锁 ¥8,000）
   - IP 孵化组 - **未配置**（红色徽章）
   - 个人账户 ¥3,200
   > "这是骋风天合的全部资金账户拓扑：1 个企业主账户 → 多个团队预算账户 → 个人账户独立。"

#### 演示企业管理员视角下的"划拨预算"

3. **去顶部空间切换器，切到"企业管理员视角"**：
   - 顶部钱包变成 ¥1,250,000
   - "当前空间绑定账户"大卡变成紫色（企业主账户）
   - 出现"📤 划拨预算给团队"按钮
4. **点"划拨预算给团队"** → 弹 modal
5. **演示给 IP 孵化组配置预算**：
   - 选目标"IP 孵化组（暂未开通预算账户）"
   - 输入金额 100000
   - 看 modal 内的箭头：企业主 ¥1,250,000 → ¥1,150,000；IP 孵化组 0 → ¥100,000
   - 点"确认划拨"
   - Toast："✅ 已划拨 ¥100,000 至 IP 孵化组"
   - 提示："IP 孵化组已开通预算账户，下次该空间下单不再回退至企业主账户"
6. **看下方资金流水**：双向流水自动写入（企业主账户 -¥100,000，IP 孵化组 +¥100,000）
7. **演示对公转账充值**（可选）：
   - 点企业主账户的"企业充值"
   - 选"对公转账（财务审核）"
   - 输入 ¥100,000 → 提交
   - Toast："对公转账凭证已提交，财务审核中（约 1 个工作日到账）"
   - **3 秒后自动模拟审核通过** → Toast："💰 对公转账已审核通过，余额到账 ¥100,000"
   > "这就是 PDF 4.2 节里说的'对公转账走财务审核流程'。"

### 业务方可能会问

- Q: "为什么不直接给团队充值？要从企业主账户划拨？"
  - A: "因为企业的钱**只能进一次平台**——开票、税务、对公账户全是企业层面的。团队预算是**内部分账**，不涉及外部支付。"
- Q: "团队预算超支怎么办？"
  - A: "下单时余额不足直接拦下（场景 3 演过了）。如果业务必须执行，团队成员要找企业管理员追加预算。"
