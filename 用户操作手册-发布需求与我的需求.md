## 1. 功能概览

「发布需求」和「我的需求」用于完成需求方从发起需求到跟踪需求进度的核心流程。

### 1.1 发布需求可以做什么

- 选择发布方式：平台套餐或自定义发布。
- 填写需求类目、标签、服务等级、制作方式、类目专属字段。
- 填写需求标题、详细描述、预算、交付时间和参考资料。
- 暂存草稿、重置表单或提交需求。
- 提交后根据发布方式进入不同后续流程。

### 1.2 我的需求可以做什么

- 查看当前用户参与的所有团队需求。
- 按团队范围、类目、状态、关键词筛选需求。
- 查看需求状态、报价数、预算、截止时间和扣款主体。
- 进入需求详情、报价对比、项目协同空间。
- 对已驳回需求进行修改并重提。

## 2. 使用前准备

### 2.1 确认当前团队

页面顶部有团队切换器。发布需求前，请确认当前团队是否正确。

常见团队包括：

| 团队 | 说明 |
|------|------|
| 个人 | 使用个人账户发布与扣款 |
| 骋风天合 · 短剧制作中心 | 使用短剧制作中心团队预算账户 |
| 骋风天合 · 市场宣传组 | 使用市场宣传组团队预算账户 |
| 骋风天合 · IP 孵化组 | 如未配置团队预算，将回退至企业主账户 |
| 企业管理员视角 | 用于管理、预算与后台操作，不建议直接发布业务需求 |

### 2.2 确认扣款主体

发布页右侧「实时摘要」会显示：

- 发布到团队
- 下单后扣款主体
- 是否存在团队未配置预算、回退企业主账户等提示

提交前请确认扣款主体符合预期。

### 2.3 准备需求资料

发布需求前建议准备：

- 需求标题
- 需求背景和详细描述
- 参考案例链接或素材链接
- 预算范围
- 期望交付时间
- 必须呈现和必须避免的内容
- 验收标准或交付格式要求

## 3. 发布需求

> 入口：左侧侧栏 → 我有需求 → 发布需求

发布需求页支持两种发布方式：

| 发布方式 | 适用场景 | 审核与后续 |
|----------|----------|------------|
| 选平台套餐 | 需求规格清晰，适合使用平台标准服务 | 提交后系统自动审核通过，并自动创建项目协同空间 |
| 自定义发布 | 需求非标，需要自定义类目、标签、预算和交付要求 | 提交后进入平台审核核定，审核通过后创建项目协同空间并撮合承制方 |

### 3.1 选择发布方式

进入发布页后，先在「发布方式」区域选择：

- 选平台套餐
- 自定义发布

默认展示「选平台套餐」。

#### 选平台套餐

适合标准化服务。用户需要先选服务类型，再选套餐档位。

操作步骤：

1. 点击「选平台套餐」。
2. 在「选平台套餐」区域选择服务类型（SPU）。
3. 选择具体套餐档位（SKU）。
4. 系统会自动带出套餐价格、交付数量、交付周期等信息。
5. 填写需求标题、详细描述、交付时间说明和参考资料。
6. 点击「提交需求」。

提交结果：

- 系统自动审核通过。
- 需求状态进入「撮合中」。
- 项目协同空间自动创建。
- 用户可在「我的需求」或「项目协同空间」中继续跟进。

#### 自定义发布

适合非标准需求。用户需要完整填写类目、标签、预算和交付信息。

操作步骤：

1. 点击「自定义发布」。
2. 选择一级类目和二级类目。
3. 选择标签体系。
4. 选择服务等级和制作方式。
5. 填写类目专属字段。
6. 填写基本信息、预算和交期。
7. 检查右侧实时摘要。
8. 点击「提交需求」。

提交结果：

- 需求进入平台审核核定环节。
- 审核通过后，系统创建项目协同空间。
- 平台撮合 3-5 家承制方报价。
- 用户可在「我的需求」中查看审核与报价状态。

### 3.2 选择需求类目

自定义发布时，需要先选择需求类目。

页面分为：

- 一级类目
- 二级类目

当前支持的一级类目包括：

| 一级类目 | 示例二级类目 |
|----------|--------------|
| 视频制作 | 短剧解说、营销短视频、宣传片等 |
| 文案服务 | 营销文案、脚本、短视频文案等 |
| 图片设计 | 海报设计、封面设计、信息图、Logo / VI 等 |
| 内容创作 | 配音、图文内容、创意策划等 |
| 资质服务 | ICP 备案、影视制作经营许可、内容审核报备、法务咨询等 |

选择一级类目后，系统会展示对应二级类目。

### 3.3 选择标签体系

选择类目后，页面会展示对应标签。标签用于帮助平台更精准地撮合承制方。

标签通常按多个维度组织，例如：

- 风格
- 题材
- 场景
- 渠道
- 交付偏好

建议尽量选择与需求高度相关的标签，避免选择过多无关标签。

### 3.4 选择服务等级和制作方式

服务等级用于表达对质量、复杂度、专业度的要求。

制作方式用于表达希望承制方采用的生产方式，可能包括人工制作、AI 辅助、特效制作、拍摄剪辑等。

注意事项：

- 服务等级通常为单选。
- 制作方式可按页面支持多选。
- 服务等级越高，预算和交付周期通常需要更充足。

### 3.5 填写类目专属字段

不同类目会出现不同表单字段。比如：

- 视频制作可能需要填写视频时长、集数、比例、平台渠道。
- 图片设计可能需要填写尺寸、用途、交付格式、数量。
- 文案服务可能需要填写字数、语气、使用场景。
- 资质服务可能需要填写主体信息、办理地区、材料准备情况。

填写时请尽量明确，减少后续沟通成本。

### 3.6 填写基本信息、预算和交期

基本信息区域包含以下字段：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 需求标题 | 是 | 一句话说明要做什么，建议控制在 40 字以内 |
| 详细描述 | 是 | 说明背景、目标、核心卖点、必须呈现/避免的元素、参考案例等 |
| 预算范围（含税） | 自定义发布必填 | 填写最低预算和最高预算 |
| 期望交付时间 | 是 | 可填写具体日期或 T+N 工作日 |
| 参考资料 | 否 | 可添加参考链接、素材链接或说明文档链接 |

平台套餐模式下，预算由 SKU 锁定，页面会隐藏自定义预算范围。

### 3.7 查看实时摘要

右侧「实时摘要」会同步展示已填写内容，包括：

- 发布方式
- 平台套餐
- 类目
- 已选标签
- 服务等级 / 制作方式
- 类目专属字段
- 预算与交期
- 发布到团队
- 下单后扣款主体

提交前建议逐项检查摘要，确认无误后再提交。

### 3.8 暂存草稿

如果暂时不提交，可点击底部「暂存草稿」。

适用场景：

- 资料未准备齐全。
- 预算或交期仍需内部确认。
- 需要等待团队成员补充素材。

再次进入发布页时，系统会尝试恢复草稿内容。

### 3.9 重置表单

点击「重置」会清空当前填写内容并恢复初始状态。

注意：重置后当前未保存内容可能丢失。请确认不再需要当前内容后再操作。

### 3.10 提交需求

点击「提交需求」后，系统会进行必要校验。

平台套餐模式需要满足：

- 已选择服务类型（SPU）
- 已选择套餐档位（SKU）
- 已填写需求标题
- 已填写详细描述
- 已填写交付时间说明

自定义发布模式需要满足：

- 已选择一级类目和二级类目
- 已选择服务等级和制作方式
- 已填写需求标题
- 已填写详细描述
- 已填写预算范围
- 已填写期望交付时间
- 预算最低值不能高于预算最高值

提交成功后，系统会跳转或提示用户进入后续跟进页面。

## 4. 我的需求

> 入口：左侧侧栏 → 我有需求 → 我的需求

「我的需求」用于查看和处理用户参与团队下的所有需求。

### 4.1 页面组成

页面主要包含：

- 发布新需求按钮
- 状态聚合卡
- 筛选工具栏
- 需求列表
- 每条需求的操作入口

### 4.2 发布新需求

点击页面右上角「发布新需求」，进入发布需求页。

### 4.3 状态聚合卡

页面顶部展示核心状态统计：

| 状态卡 | 说明 |
|--------|------|
| 全部 | 当前筛选范围内所有需求 |
| 审核中 | 已提交，等待平台审核核定 |
| 撮合中 | 已通过审核，平台正在撮合承制方报价 |
| 待选承制方 | 已收到报价，需要需求方选择承制方 |
| 进行中 | 已下单，承制方正在制作或交付中 |

状态卡可点击。点击后列表只展示对应状态的需求。

页面还会展示辅助状态：

- 已驳回
- 已取消
- 已完成

点击辅助状态可查看对应归档需求。

### 4.4 按团队范围筛选

工具栏中的「团队范围」支持：

| 筛选项 | 说明 |
|--------|------|
| 仅当前团队 | 只展示当前顶部团队下的需求 |
| 所有团队（我参与的） | 展示当前用户参与的全部团队需求 |

如果找不到某条需求，可先切换为「所有团队（我参与的）」。

### 4.5 按类目筛选

「类目」筛选支持按一级类目查看需求，例如：

- 视频制作
- 文案服务
- 图片设计
- 内容创作
- 资质服务

### 4.6 按关键词搜索

搜索框支持输入：

- 需求名
- 需求号

输入后列表会实时过滤。

### 4.7 需求列表字段说明

| 字段 | 说明 |
|------|------|
| 需求 | 展示需求标题、需求号、创建时间；标题可点击进入需求详情 |
| 类目 | 展示需求所属服务类目 |
| 所属团队 | 展示需求归属团队及扣款主体 |
| 状态 | 展示当前需求状态及状态说明 |
| 报价数 | 展示已收到报价数量 |
| 预算 / 截止 | 展示预算金额与期望交付时间 |
| 操作 | 展示详情、报价对比、项目空间、修改重提等操作 |

### 4.8 常见操作按钮

| 按钮 | 出现条件 | 作用 |
|------|----------|------|
| 详情 | 所有需求 | 进入需求详情页 |
| 对比报价 | 待选承制方状态 | 查看承制方报价并选择合作方 |
| 项目空间 | 需求已创建项目空间 | 进入项目协同空间 |
| 修改并重提 | 已驳回状态 | 修改需求后重新提交 |
| 归档 | 已完成或已取消状态 | 查看归档信息 |

## 5. 需求详情

> 入口：我的需求 → 点击需求标题或「详情」

需求详情页用于查看单条需求的完整信息。

### 5.1 页面展示内容

需求详情页包含：

- 需求号
- 需求标题
- 当前状态
- 服务类目
- 所属团队
- 预算
- 需求类型
- 创建时间
- 期望截止时间
- 扣款主体
- 报价数
- 承制方
- 需求详细描述
- 处理记录
- 关联项目空间
- 下一步建议

### 5.2 可执行操作

根据需求状态不同，详情页可能展示以下操作：

- 返回列表
- 对比报价
- 进入项目空间
- 修改并重提

### 5.3 处理记录说明

处理记录会展示需求流转过程，例如：

1. 需求已提交
2. 平台核定通过
3. 项目空间已创建
4. 订单已生成

用户可通过处理记录快速判断需求当前处在哪个阶段。

## 6. 需求状态说明

| 状态 | 含义 | 用户建议操作 |
|------|------|--------------|
| 审核中 | 需求已提交，平台正在核定 | 等待审核结果，可准备补充资料 |
| 撮合中 | 审核已通过，平台正在撮合承制方 | 等待报价，可进入项目空间查看需求沉淀 |
| 待选承制方 | 已收到报价，需要选择承制方 | 进入报价对比，选择合适承制方 |
| 进行中 | 已下单，承制方正在制作 | 进入项目空间跟进素材、版本、沟通和验收 |
| 已完成 | 需求已验收并结算 | 查看归档、钱包流水或项目空间记录 |
| 已取消 | 需求已撤回或取消 | 查看归档，如需重新发起请发布新需求 |
| 已驳回 | 平台审核未通过 | 查看原因，修改后重新提交 |

### 6.1 平台套餐状态流转

平台套餐通常流转为：

1. 选择平台套餐
2. 提交需求
3. 系统自动审核通过
4. 自动创建项目协同空间
5. 进入撮合中
6. 收到报价后进入待选承制方
7. 选择承制方后进入履约

### 6.2 自定义发布状态流转

自定义发布通常流转为：

1. 填写自定义需求
2. 提交需求
3. 进入审核中
4. 平台核定通过
5. 自动创建项目协同空间
6. 平台撮合承制方报价
7. 收到报价后进入待选承制方
8. 选择承制方后进入履约

## 7. 常见操作场景

### 7.1 发布一个平台套餐需求

1. 进入「发布需求」。
2. 选择「选平台套餐」。
3. 选择服务类型和套餐档位。
4. 填写需求标题和详细描述。
5. 填写交付时间说明。
6. 检查右侧实时摘要。
7. 点击「提交需求」。
8. 在「我的需求」中查看新需求状态。
9. 点击「项目空间」进入协同空间跟进。

### 7.2 发布一个自定义需求

1. 进入「发布需求」。
2. 选择「自定义发布」。
3. 选择一级类目和二级类目。
4. 选择标签。
5. 选择服务等级和制作方式。
6. 填写类目专属字段。
7. 填写标题、描述、预算、交期和参考资料。
8. 检查右侧实时摘要。
9. 点击「提交需求」。
10. 在「我的需求」中等待平台审核。

### 7.3 查看某条需求进度

1. 进入「我的需求」。
2. 使用团队范围、类目或关键词找到需求。
3. 点击需求标题或「详情」。
4. 在需求详情页查看状态、处理记录和下一步建议。

### 7.4 处理待选承制方需求

1. 进入「我的需求」。
2. 点击「待选承制方」状态卡。
3. 找到需要处理的需求。
4. 点击「对比报价」。
5. 查看承制方报价和方案。
6. 选择承制方并生成订单。
7. 后续进入项目协同空间进行履约跟进。

### 7.5 进入项目协同空间

可从以下入口进入：

- 我的需求 → 项目空间
- 需求详情 → 进入项目空间
- 左侧侧栏 → 协同 → 项目协同空间

项目协同空间用于跟进：

- 需求详情
- 素材包
- 交付与版本
- 沟通记录
- 验收记录

### 7.6 修改被驳回的需求

1. 进入「我的需求」。
2. 点击辅助状态「已驳回」。
3. 找到被驳回需求。
4. 点击「修改并重提」。
5. 根据驳回原因修改内容。
6. 再次提交需求。

## 8. 常见问题

### Q1：为什么我发布需求时显示当前团队不能发布？

可能当前处于「企业管理员视角」。该视角主要用于成员、权限、预算和后台管理，不建议直接发布业务需求。请切换到具体业务团队后再发布。

### Q2：平台套餐和自定义发布有什么区别？

平台套餐是标准化服务，价格和交付规格由套餐锁定，提交后系统自动审核通过。自定义发布适合非标需求，需要平台人工核定后再撮合承制方。

### Q3：为什么平台套餐不用填写预算？

平台套餐的预算由所选 SKU 固定，系统会按套餐价格生成预算和条款快照，因此不需要手动填写预算范围。

### Q4：提交后为什么状态是审核中？

自定义发布需要平台核定需求信息、预算、交期和类目匹配情况。核定通过后才会进入撮合流程。

### Q5：为什么有些需求没有项目空间入口？

项目空间通常在需求通过审核后创建。如果需求仍在审核中、被驳回或已取消，可能暂未创建项目空间。

### Q6：什么情况下会出现「对比报价」？

当需求已收到承制方报价，并进入「待选承制方」状态时，会出现「对比报价」按钮。

### Q7：为什么在我的需求里找不到某条需求？

请检查：

1. 团队范围是否选为「仅当前团队」。
2. 是否使用了类目筛选。
3. 是否输入了关键词。
4. 是否点选了某个状态卡。

如不确定，可切换为「所有团队（我参与的）」并清空筛选条件。

### Q8：预算范围应该怎么填写？

建议填写真实可接受范围。最低预算不要高于最高预算。预算过低可能影响承制方报价质量或撮合效率。

### Q9：参考资料可以不填吗？

可以不填，但建议提供参考案例、素材链接或说明文档。参考资料越明确，承制方理解需求越准确。

### Q10：需求完成后还能查看吗？

可以。已完成需求会进入归档状态，可在「我的需求」的辅助状态「已完成」中查看，也可通过项目协同空间查看履约记录。
