# 天合短剧剪辑接单平台 —— 项目介绍（投资人版）

> 面向投资人汇报用 · 基于一期 MVP 产品说明

---

## 一、项目概述

**天合接单平台** 是一个面向短剧、AI 漫剧等内容制作领域的 **项目制接单与协同交付平台**。平台连接内容制作方（学员/剪辑师）与项目需求方（甲方），实现从项目发布、接单、交付、审核到验收、结算的全流程数字化管理。

**核心价值**：解决项目制内容交付中「项目散落、版本混乱、收益不清」的痛点，为运营方提供可规模化、可追溯的一体化管理能力。

**交付形态**：学员端 Web/H5 + 后台管理 Web，一期 MVP 已完成核心闭环。

---

## 二、市场与痛点

### 2.1 行业背景

短剧、AI 漫剧等短视频内容快速增长，剪辑、调色等制作需求大量释放。项目制交付是主流模式：按项目发布、按集数计费、多人协作完成。

### 2.2 核心痛点

| 痛点 | 现状 | 影响 |
|------|------|------|
| 项目与订单分散 | 依赖表格、群聊管理，状态流转靠人工 | 责任不清、进度难追踪 |
| 版本与修改难追溯 | 素材与成品无统一版本管理 | 多轮修改难以复盘、纠纷风险高 |
| 运营无法规模化 | 缺少项目台账、产能与人员管理 | 人力成本高、扩张受限 |
| 收益口径不一致 | 榜单、收益、对账缺乏统一账本 | 激励不透明、结算易出错 |

### 2.3 目标用户

- **学员/接单方**：剪辑师、调色师等制作人员，通过平台接单获得收益
- **甲方**：内容出品方，发布项目需求、验收交付
- **运营**：平台运营方，负责项目发布、人员调配、审核协调
- **审核**：负责交付质量把关
- **财务**：负责对账、结算、报表

## 三、产品核心功能

### 3.1 业务流程闭环

```
发布项目 → 接单（抢单/报名）→ 交付 → 审核 → 验收 → 入账 → 对账/结算
```

全流程在线完成，状态可追踪、操作可留痕。

### 3.2 学员端功能

| 模块 | 功能说明 |
|------|----------|
| 接单大厅 | 按短剧/AI漫剧/短视频/综艺分类展示项目，支持筛选、排序、一键接单或报名 |
| 项目详情 | 查看需求、单价、集数、截止时间，一键接单（抢单模式）或提交报名（报名模式） |
| 我的项目 | 查看所有订单及报名状态（进行中/已提交/待审核/已驳回等），一键进入项目空间 |
| 项目协同空间 | 下载素材、上传交付、查看修改意见、沟通留痕，支持多轮修改与版本管理 |
| 账户与收益 | 累计收益、可提现余额、完成订单数，收益与财务账本一致 |
| 天梯榜单 | 总接单量、总收益、近期活跃度多维排行，激励透明 |

### 3.3 后台管理功能

| 模块 | 功能说明 |
|------|----------|
| 项目管理 | 新建、编辑、发布、下架、关闭、归档；项目台账、订单统计、改派/延期/关闭 |
| 学员管理 | 入驻审核、人才库、技能标签、等级、并发上限、状态管理 |
| 任务审核 | 审核队列（临期优先）、查看交付版本与修改记录、人工通过/退回 |
| 财务对账 | 账本、入账（验收后）、扣款调整、结算标记、导出报表 |
| 数据看板 | 短剧整体概览、项目状态看板、进审时间线、订单与收益统计 |

### 3.4 接单模式

- **抢单**：先到先得，满足资格即可直接接单
- **报名**：提交报名后由运营审核，通过后自动创建订单和协同空间

## 四、核心业务规则

### 4.1 入账与收益

- **入账节点**：订单验收后（Accepted）计入收益
- **收益口径**：Σ（已验收订单金额）− 扣款调整
- **榜单与收益**：支持公开展示收益，用于激励

### 4.2 订单状态流转

```
待接单 → 已接单(进行中) → 已提交 → [退回修改] → 审核通过 → 已验收(入账) → 已关闭
```

### 4.3 项目状态

```
草稿 → 已发布 → 已满/已截止 → 已关闭 → 已归档
```

## 五、技术架构

- **前端**：React + Ant Design，Less 样式，多页面 HTML 架构
- **数据**：LocalStorage 本地存储（MVP 阶段），支持后续对接后端 API
- **图表**：AntV G2 用于数据看板可视化
- **交付形态**：纯前端 Web 应用，可部署至静态托管或接入后端服务

---

## 六、MVP 验收口径

| 指标 | 说明 |
|------|------|
| 接单转化率 | 接单成功人数 / 项目详情 UV |
| 准时交付率 | 提交时间 ≤ 截止时间的订单数 / 已接单订单数 |
| 验收闭环率 | 已验收订单数 / 已提交订单数 |
| 平均交付周期 | 首次提交时间 − 接单时间 |
| 平均审核周期 | 审核结论时间 − 提交时间 |
| 收益口径 | 与财务账本一致，订单不重复计入 |

---

## 七、产品亮点

1. **全流程闭环**：从发布到结算一站式完成，减少人工协调成本
2. **可追溯**：素材、交付、修改意见、沟通全程留痕，支持审计与复盘
3. **收益透明**：学员可见收益与榜单，激励清晰；财务与榜单口径一致
4. **可运营**：后台支持项目、学员、审核、财务、数据全链路管理
5. **扩展性**：支持抢单/报名双模式，可配置并发上限、榜单规则等
