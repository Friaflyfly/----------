## 1. 需求背景

平台当前已经具备自定义需求发布、需求审核、资金冻结、项目协同、验收和发票后分账的基础链路，但 AIGC 服务中存在大量高频、规格清晰、价格相对标准的需求，例如 AI 图片制作、电商套图、商品详情页、信息流短视频、本地生活短视频、剧情号短视频和品牌宣传短片。

如果所有需求都走自定义发布，用户需要填写过多字段、等待平台审核和报价撮合，转化路径较长。标准产品包页面的目标是把这类服务做成“可浏览、可比较、可直接购买”的商品化入口，降低购买决策成本，同时沿用平台现有资金托管、订单履约、发票和结算体系。

## 2. 需求目标

1. 建立“标准包购买”入口，支持用户浏览标准化产品包、查看详情弹窗、查看样例占位和价格信息。
2. 建立购买配置弹窗，支持用户选择档位、数量、规格、视频时长、画幅比例、增项，并编辑预置的需求描述模板。
3. 建立确认订单链路，购买后生成平台套餐需求单、订单、冻结单和项目协同空间。
4. 购买时明确展示付款账户、收款主体、订单金额、资金动作和余额不足处理。
5. 保持与资金中心一致：支付后先冻结预算，不立即结算给承制方；验收后等待承制方上传发票，平台财务校验通过后再分账。
6. 为后续后台运营管理标准产品包、价格、上下架、样例素材和承制方承接规则预留对象模型。

## 6. 对象关系

```text
SPU
└── SKU
    └── PurchaseConfig
        └── Demand（PLATFORM_SKU）
            ├── Order
            │   ├── FreezeOrder
            │   ├── ProjectSpace
            │   └── Invoice / Settlement
            └── Bill / TransactionFlow / FundFlow
```

生命周期联动：

- SPU/SKU 下架不影响历史需求和历史订单，历史订单读取购买时的 `sku_snapshot`。
- SKU 调价不影响已购买订单，已购买订单锁定购买时交易价。
- 购买失败不得生成有效订单；若已生成冻结单但订单创建失败，需触发解冻或补偿。
- 订单取消时，冻结单进入 cancelled，资金回到付款账户。
- 订单验收通过后，不立即分账；需等待承制方上传发票并由平台财务校验通过。

## 7. 总体流程

### 7.1 主流程

1. 用户进入标准产品包页面。
2. 用户浏览产品包卡片。
3. 用户点击“查看详情”，打开产品包详情弹窗。
4. 用户点击“立即购买”，打开购买配置弹窗。
5. 用户选择档位、数量、规格、增项，并按预置模板补充或修改需求描述，系统实时计算价格。
6. 用户点击“立即购买”，打开确认订单弹窗。
7. 系统展示付款账户、收款主体、当前余额、订单金额、资金动作。
8. 用户点击“确认付款并提交”。
9. 系统校验余额。
10. 系统冻结付款账户对应金额。
11. 系统生成平台套餐需求单。
12. 系统生成订单。
13. 系统创建项目协同空间。
14. 承制方履约交付。
15. 需求方确认验收。
16. 承制方上传发票。
17. 平台财务校验发票。
18. 系统完成分账结算。

### 7.2 余额不足流程

1. 用户确认订单时，系统计算当前可用余额。
2. 若余额不足，确认按钮置灰或点击后提示不足。
3. 页面提供“去充值 / 切换付款账户 / 返回修改配置”入口。
4. 用户充值后返回购买页，系统重新读取余额。
5. 余额充足后继续冻结和下单。

## 8. 状态机

### 8.1 订单状态

| 状态 | 含义 | 可操作项 |
|---|---|---|
| `pending_payment` | 待付款 | 继续付款、取消 |
| `payment_frozen` | 已冻结预算 | 创建项目空间、等待承接 |
| `in_progress` | 承制方履约中 | 查看项目空间、补充素材 |
| `review` | 待验收 | 确认验收、驳回修改 |
| `waiting_invoice` | 已验收，待承制方上传发票 | 等待发票 |
| `invoice_reviewing` | 发票校验中 | 查看发票状态 |
| `pending_settlement` | 待分账 | 查看资金状态 |
| `completed` | 已完成已结算 | 查看归档、查看账单 |
| `cancelled` | 已取消 | 查看退款/解冻记录 |

## 9. 页面设计

### 9.1 标准产品包购买页

菜单名称：**标准包购买**。

页面结构：

1. 顶部全局导航：团队/账户/资金中心入口。
2. Hero 区：说明“AIGC标准化产品服务包”、购买优势、托管说明和关键规则。
3. 分类 Tabs：全部、图片制作、电商设计、短视频制作、本地生活等。
4. 首购权益说明条：展示先锁价、资料后补、可转定制等购买权益。
5. 产品包卡片区：展示产品包卡片。
6. 详情弹窗：查看规格参数、素材上传要求、对标示例和交付要求。
7. 购买配置弹窗：选择具体购买参数并填写需求描述。
8. 确认订单弹窗：确认资金和订单信息。

- “立即购买”打开购买配置弹窗，不在右侧常驻展示购买表单。
- 卡片需预留图片或视频样例封面位置，后续可替换为真实作品素材。

模板规则：

- 模板是可编辑内容，不是只读说明。
- 用户未修改模板时，系统不得把提示模板直接作为真实需求提交。
- 用户修改后，提交修改后的需求描述。
- 需求描述后续作为承制方理解需求、验收和争议处理的依据之一。

价格计算：

```text
合计金额 = 基础档位价格 × 数量 + 时长差价 × 数量 + 增项费用
```

[财务口径待明确] 增项费用、加急费用、源文件费用、促销折扣的最终定价规则需由运营/财务确认。

| 字段 | 说明 |
|---|---|

按钮：

## 12. 业务规则

### 12.1 购买主体规则

- 当前全局账户上下文决定付款账户。
- 个人账户购买时，付款账户为个人账户。
- 团队账户购买时，付款账户为团队预算账户。
- 企业管理员视角原则上不直接购买产品包；如需要代下单，需切换到具体团队。[待确认]

### 12.2 收款主体规则

- 标准产品包购买确认页必须展示收款主体：长沙骋风天合文化科技有限公司。
- 收款主体不随团队切换改变。
- 发票开具主体、税率、发票内容与收款主体关系需财务确认。[财务口径待明确]

### 12.3 资金规则

- 购买确认后先冻结订单金额。
- 冻结成功后才生成有效订单。
- 冻结失败时不得生成有效订单。
- 冻结资金在订单完成前不可用于其他消费。
- 取消订单、审核异常或平台无法履约时，应解冻回原付款账户。
- 验收后仍不立即分账，需等待承制方上传发票并由平台财务校验通过。

### 12.4 商品快照规则

- 下单时必须保存 SKU 快照。
- 快照包括产品包名称、档位、价格、购买配置、需求描述、交付内容、修改次数、交付周期、条款版本。
- 后续 SPU/SKU 调整不影响历史订单。

### 12.5 价格规则

- 页面展示交易价和划线价。
- 交易价为实际购买金额计算基础。
- 划线价仅为展示字段，不参与结算。
- 卡片展示最低交易价和划线价；详情弹窗展示产品包对应的单价或多档价格方案。
- 促销权益、首购加赠和买赠规则需运营配置。[运营口径待明确]

## 13. 异常场景与处理

| 场景 | 处理 |
|---|---|
| 余额不足 | 展示差额，引导充值或切换账户 |
| 产品包下架 | 卡片不可购买；已下单订单不受影响 |
| SKU 调价 | 未下单按新价，下单后按快照价 |
| 重复点击确认付款 | 按幂等请求号防重复冻结和重复创建订单 |
| 冻结成功但订单创建失败 | 自动重试；失败后解冻并提示 |
| 订单创建成功但项目空间创建失败 | 订单保留，项目空间补偿创建 |
| 素材未上传 | 允许先下单，项目空间内补充素材；是否强制上传按产品包配置 |
| 承制方未及时上传发票 | 订单停留在待上传发票，平台可提醒或人工处理 |
| 发票校验驳回 | 承制方重新上传，订单不分账 |
| 用户取消订单 | 按订单状态判断是否允许取消和解冻，[法务口径待明确] |

## 18. 验收标准

### 18.1 页面验收

1. 用户进入标准包购买页，可以看到 Hero 区、规则文案、分类 Tabs、首购权益说明条和产品包卡片。
2. 产品包卡片展示交易价和划线价，价格不在按钮区域重复展示。
3. 产品包卡片只展示“查看详情”和“立即购买”两个主操作。
4. 点击“查看详情”打开详情弹窗，展示主要场景、规格参数、价格方案、素材上传要求、对标示例和交付要求。
5. 详情弹窗内产品名称只出现一次，场景描述只出现一次，不展示“适用用户”和“价格因子”。
6. 点击“立即购买”打开购买配置弹窗。
7. 购买配置弹窗展示需求描述模板，用户可删除、增加或修改模板内容。
8. 修改数量、时长、规格和增项后，合计金额实时变化。
9. 点击购买后打开确认订单弹窗，展示付款账户、收款主体、订单金额和资金动作。

### 18.2 购买链路验收

1. 余额充足时，点击确认付款生成需求单、订单和冻结单。
2. 余额不足时，系统提示差额，不生成有效订单。
3. 下单成功后，在我的需求中可以看到 PLATFORM_SKU 类型需求。
4. 下单成功后，在项目空间中可以继续履约。
5. 下单成功后，在资金中心能看到对应冻结/账单/流水。
6. 验收后订单进入待上传发票，不立即分账。
7. 承制方上传发票后订单进入发票校验中。
8. 财务校验通过后订单进入待分账，再完成分账。

### 18.3 异常验收

1. 重复点击确认付款不会重复冻结。
2. SKU 下架后不可新购，但历史订单可查看。
3. 项目空间创建失败时，订单可通过补偿任务补建空间。
4. 发票驳回后承制方可重新上传。
