## 1. 平台后台概览

后台管理系统支持以下核心角色：

| 角色 | 主要职责 |
|------|----------|
| 运营 | 项目发布/下架、报名审核、进度跟进 |
| 审核 | 查看交付内容、人工通过/退回 |
| 财务 | 对账、扣款调整、结算标记 |
| 管理员 | 学员入驻审核、权限与数据管理 |

后台入口位于各页面**左侧侧栏**的「后台管理」分组，包含：

- ✅ 任务审核
- 📋 项目管理
- 👥 学员管理
- 💰 财务对账
- 📈 数据看板

## 2. 项目管理：发布与维护项目

> 入口：侧栏 → **项目管理**（`ProjectManagement.html`）

### 2.1 项目列表（台账）

列表展示所有项目，包含字段：

| 字段 | 说明 |
|------|------|
| 项目ID | 系统生成唯一编号 |
| 项目名称 | 可点击进入项目详情 |
| 分类 | 短剧 / AI漫剧 / 短视频 / 综艺 |
| 城市 | 长沙 / 西安 |
| 项目状况 | 在制中 / 已交付 / 待进 |
| 单价 | 元/集 |
| 集数 | 总集数 |
| 截止时间 | 项目交付截止时间 |
| 名额/已接单数 | 可接名额与当前已接数量 |
| 项目状态 | 草稿/已发布/已满已截止/已关闭/已归档 |
| 负责人/统筹 | 项目统筹人员 |
| 订单统计 | 进行中/待审核/待修改/待验收/已验收数量 |

### 2.2 筛选

支持多条件组合筛选：

- 状态、城市、项目状况、项目阶段（精剪中/一审等）
- 分类、接单模式、难度、参与团队、统筹人员
- 截止时间区间
- 关键字搜索（项目ID / 名称）

### 2.3 新建项目

1. 点击右上角「**新建项目**」
2. 跳转到项目编辑页（`ProjectEdit.html`）
3. 填写项目信息后保存或发布

**项目编辑页包含以下 Tab：**

#### 基础信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 项目名称 | ✅ | ≤50字 |
| 内容分类 | ✅ | 短剧/AI漫剧/短视频/综艺 |
| 项目标签 | ✅ | 霸总/穿越/男频/女频等 |
| 内容类别 | ✅ | 题材类型 |
| 难度 | ✅ | 精品 / 工厂 |
| 参与团队 | ✅ | 实训 / 线下 / 线上 |
| 城市 | ✅ | 长沙 / 西安 |
| 项目状况（一级） | ✅ | 在制中 / 已交付 / 待进 |
| 项目阶段（二级） | ✅ | 精剪中 / 内部一审 / 内部二审 / 导演审 / 监制审 / 平台审 / 审核完成 |

#### 成本与预算

| 字段 | 必填 | 说明 |
|------|------|------|
| 预算金额 | ✅ | 项目预算，对外不展示 |
| 支出金额 | — | 已发生支出，可后置填写 |
| 项目备注 | — | 运营内部备注 |
| 甲方反馈 | — | 结束后回填 |

#### 交付与排期

| 字段 | 必填 | 说明 |
|------|------|------|
| 交付周期 | ✅ | 天数 |
| 截止时间 | ✅ | 项目截止日期 |
| 集数 | ✅ | 总集数 |

#### 接单配置

| 字段 | 必填 | 说明 |
|------|------|------|
| 接单模式 | ✅ | 抢单（先到先得）/ 报名（运营审核） |
| 名额 | ✅ | 可接单人数上限 |
| 报名审核策略 | — | 仅报名模式：自动通过 / 运营审核 |
| 指定候选人 | — | 限制可见范围 |

#### 需求与素材

| 字段 | 必填 | 说明 |
|------|------|------|
| 项目需求文本 | ✅ | 核心需求描述 ≤5000字 |
| 素材说明 | — | 补充说明 |
| 素材包 | — | 上传脚本/源视频/参考样片等 |

#### 统筹人员

选择该项目负责的统筹人员（用于台账与责任归属）。

### 2.4 发布/下架

- **发布**：点击操作列「发布」按钮，项目状态变为「已发布」，学员端可见
- **下架**：点击「下架」，项目状态变为「已关闭」，学员端不可见
- **归档**：项目完结后点击「归档」，状态变为「已归档」

> ⚠️ 发布/下架操作**立即生效**，影响学员端可见性。

### 2.5 编辑项目

点击项目行「编辑」或直接点击项目名称进入详情，可修改项目信息后保存。

### 2.6 进入项目空间

点击项目ID链接，或操作列「进入空间」，可进入该项目的协同工作空间，查看所有订单状态与历史。

### 2.7 报名审核（报名模式项目）

当项目配置为「报名模式」时：

1. 点击订单统计列的「报名」数字，展开报名列表
2. 查看报名人昵称、申请时间、备注
3. 点击「**通过**」审核通过：
   - 系统自动创建订单，状态进入 InProgress
   - 自动创建项目协同空间
4. 点击「**驳回**」：
   - **必须填写驳回原因**
   - 报名状态更新为已驳回，学员可查看原因

## 3. 学员管理：人才库与入驻审核

> 入口：侧栏 → **学员管理**（`StudentManagement.html`）

### 3.1 入驻审核队列

新学员注册后需经过入驻审核，才能加入人才库并开始接单。

**审核操作：**

1. 查看待审核学员列表（昵称、申请时间、技能方向等）
2. 点击「**通过**」：生成人才档案，初始状态为「在训」
3. 点击「**驳回**」：需填写驳回原因（必填）

### 3.2 人才库

展示所有已入驻学员的综合信息：

| 字段 | 说明 |
|------|------|
| 学员ID / 昵称 | 唯一标识 |
| 状态 | 在训 / 可接单 / 暂停 / 冻结 |
| 技能标签 | 剪辑师 / 调色师 / 特效师等 |
| 等级 | 技能等级 |
| 并发占用/上限 | 当前进行中订单数 / 最大并发数 |
| 近30天接单量 | 活跃度参考 |
| 准时率 | 准时提交比例 |
| 通过率 | 审核通过比例 |
| 修改轮次均值 | 平均退回修改次数 |

### 3.3 编辑学员信息

点击学员行「编辑」，可修改：

- **技能标签**：调整技能方向
- **等级**：调整技能等级
- **并发上限**：调整个人最大并发订单数

### 3.4 状态管理

| 操作 | 效果 |
|------|------|
| 暂停 | 学员无法接新订单（进行中订单不受影响） |
| 冻结 | 账号完全禁用 |
| 解冻 | 恢复正常状态 |

> ⚠️ 暂停/冻结会立即影响学员的接单按钮可见性。

### 3.5 导出

点击「导出」，可导出当前学员列表数据为报表。

## 4. 任务审核：交付质量审核

> 入口：侧栏 → **任务审核**（`ReviewQueue.html`）

### 4.1 审核队列

展示所有待审核的交付记录：

| 字段 | 说明 |
|------|------|
| 订单号 | 可点击进入审核详情 |
| 项目名称 | 所属项目 |
| 接单人 | 学员姓名 |
| 第几集 | 本次提交的集数 |
| 是否定稿 | 是否为最终版本 |
| 版本号 | 如 V1-1、V2-1 |
| 提交审核时间 | 提交时间 |
| 等待时长 | 已等待的审核时间（超24小时标红） |
| 截止时间 | 项目截止时间（超期标红） |

**排序规则**：临期优先 + 等待时长，优先处理即将超期的订单。

### 4.2 进入审核详情

点击「**进入审核**」按钮，跳转审核详情页（`ReviewDetail.html`）。

### 4.3 审核操作

审核详情页展示：

- 交付版本列表（历史版本与修改记录）
- AI 预审结果（风险等级 / 命中标签，仅供参考，不阻断流程）
- 修改历史记录

**人工审核：**

#### 审核通过

1. 查看交付内容，确认质量达标
2. 点击「**审核通过**」
3. 系统弹窗确认
4. 订单状态更新为「审核通过（Approved）」
5. 结论同步至协同空间，写入审计日志

#### 退回修改

1. 点击「**退回修改**」
2. **必须填写**：
   - 退回原因（枚举选择）
   - 修改点说明（文本，具体描述修改要求）
3. 点击确认
4. 订单状态更新为「退回修改（NeedRevision）」
5. 学员在协同空间可查看修改意见

> ⚠️ 退回原因和修改点为必填，无法空提交。

## 5. 财务对账：账本与结算

> 入口：侧栏 → **财务对账**（`FinanceManagement.html`）

### 5.1 统计概览

页面顶部展示汇总数据：

| 指标 | 说明 |
|------|------|
| 项目数 | 账本中的项目总数 |
| 验收订单数 | 已验收订单总数 |
| 应付金额 | 所有已验收订单金额之和 |
| 扣款调整 | 录入的扣款调整总额 |
| 已结算金额 | 已标记结算的金额 |
| 未结算金额 | 尚未结算的余额 |

### 5.2 账本列表

按项目分组展示，每行代表一个项目：

| 字段 | 说明 |
|------|------|
| 项目名称 | 项目标识 |
| 金额统计 | 该项目所有已验收订单金额之和 |
| 订单总数 | 点击可查看该项目所有订单明细 |
| 验收时间（入账） | 最早入账时间 |
| 扣款调整 | 累计扣款金额 |
| 结算状态 | 未结算 / 已结算（可下拉切换） |
| 结算月份 | 标记已结算时自动记录当前月份 |

### 5.3 筛选

- **结算状态**：未结算 / 已结算
- **验收时间区间**：按入账时间筛选

### 5.4 查看订单明细

点击「X 单」链接，在右侧抽屉中查看该项目下所有订单列表：

- 订单号、接单人、接单时间、截止时间、订单状态、订单金额、扣款金额

### 5.5 扣款调整（核心操作）

**扣款针对单笔订单进行录入：**

1. 点击目标项目行右侧「**扣款调整**」按钮
2. 弹窗展示该项目下所有订单列表
3. 在每笔订单对应的「扣款金额」列中输入扣款额（不可超过该订单金额）
4. 在底部填写「**扣款原因**」（必填）
5. 点击「确认保存」

**扣款影响范围：**
- 该订单的实付金额减少
- 学员「账户与收益」中该订单金额同步更新
- 天梯榜单收益数据同步更新

> ⚠️ 扣款原因为必填，确保留痕可追溯。

### 5.6 标记结算

线下打款完成后，在平台标记结算状态：

1. 找到对应项目行
2. 点击「结算状态」列的下拉框
3. 选择「**已结算**」
4. 系统自动记录结算月份
5. 学员端收益状态同步更新为「已结算」

### 5.7 导出对账报表

点击页面右上角「**导出对账报表**」，导出当前视图的汇总与明细数据。

### 5.8 入账规则说明

- **入账节点**：订单状态变为「已验收（Accepted）」时，自动计入未结算收益
- **扣款**：由财务手动录入，录入后影响学员收益与榜单
- **结算**：线下打款后在平台标记已结算，同步更新学员端显示

## 6. 数据看板：运营数据总览

> 入口：侧栏 → **数据看板**（`DataBoard.html`）

数据看板提供短剧项目的整体运营视图，数据实时读取项目数据。

### 6.1 整体概览（KPI 卡片）

| 指标 | 口径说明 |
|------|----------|
| 2026年短剧总数量 | 所有项目数量 |
| 短剧总制作集数 | 所有订单集数之和 |
| 待进集数 | 总制作集数 − 已交付集数 |

### 6.2 短剧整体情况（饼图）

按「项目状况 + 城市」分组展示项目数量分布，如：
- 在制中 (长沙)
- 已交付 (西安)
- 待进 (长沙) 等

### 6.3 短剧进度集数分布（柱状图）

展示三个状态下的总集数对比：待进、已交付、在制中。

### 6.4 短剧项目情况（看板）

横向看板，每列对应一个「状况 + 城市」组合，列内为项目卡片。点击项目卡片可跳转至项目详情。

### 6.5 项目完成进度（看板）

按制作阶段分列展示项目进度：
- 精剪中 / 内部一审 / 内部二审 / 导演审 / 监制审 / 平台审 / 审核完成

### 6.6 短剧具体完成进度（卡片网格）

每个项目一张进度卡片，展示：
- 项目名（链接至详情）
- 当前制作阶段 + 城市
- 进度条（已交付集数 / 总集数）

### 6.7 短剧进审时间（时间线）

- **表格**：列出所有订单的接单时间、截止时间、接单人、状态
- **甘特图**：直观展示每个订单从接单到截止的时间跨度（有订单数据时显示）

## 7. 状态与流转速查

### 订单状态流转

```
待接单
  ↓ 学员接单（抢单）/ 报名审核通过
已接单（进行中）
  ↓ 学员上传交付
已提交
  ↓ 审核退回
退回修改  →  (学员修改后重新上传) →  已提交
  ↓ 审核通过
审核通过
  ↓ 申请验收，甲方确认
已验收（收益入账）
```

### 项目状态流转

```
草稿 → 已发布 → 已满/已截止 → 已关闭 → 已归档
```

### 学员状态流转

```
待入驻审核 → 在训 → 可接单 ⇌ 暂停 / 冻结 → 退出
```

### 报名状态流转

```
待审核 → 已通过（自动创建订单）
       → 已驳回（通知学员原因）
       → 已取消（学员主动撤回）
```

## 8. 常见操作场景

### 场景一：新项目上线完整流程

1. 侧栏 → 项目管理 → 「新建项目」
2. 填写基础信息、交付排期、接单配置、需求与素材
3. 选择接单模式（抢单 / 报名）
4. 保存为草稿，检查无误后点击「发布」
5. 项目出现在学员接单大厅，学员可接单

---

### 场景二：处理报名审核

1. 侧栏 → 项目管理 → 找到目标项目
2. 点击「报名数」展开报名列表
3. 查看报名人信息，决定通过或驳回
4. 通过：系统自动创建订单和协同空间
5. 驳回：填写驳回原因，学员收到通知

---

### 场景三：处理一批审核任务

1. 侧栏 → 任务审核
2. 列表按临期优先排序，优先处理临期订单
3. 点击「进入审核」查看交付内容
4. 质量达标 → 「审核通过」
5. 需修改 → 「退回修改」，填写原因和修改点
6. 返回队列，处理下一条

---

### 场景四：月末财务结算

1. 侧栏 → 财务对账
2. 筛选「未结算」项目
3. 核对应付金额与扣款情况
4. 如有扣款 → 点击「扣款调整」→ 逐笔填写扣款金额和原因 → 保存
5. 线下打款完成后，回到列表 → 点击结算状态下拉 → 选「已结算」
6. 导出对账报表存档

---

### 场景五：新学员入驻审核

1. 侧栏 → 学员管理
2. 切换到「入驻审核」Tab
3. 查看待审核学员信息
4. 通过 → 学员进入人才库（状态：在训）
5. 如需调整技能标签或等级 → 人才库 → 点击「编辑」

---

### 场景六：调整学员并发上限

1. 侧栏 → 学员管理 → 人才库
2. 找到目标学员 → 点击「编辑」
3. 修改「并发上限」数值
4. 保存后立即生效，影响该学员可接单数量
