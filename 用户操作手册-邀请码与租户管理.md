## 1. 体系概览

平台采用**统一身份入口**，接单平台与 AIGC 平台共用同一套账号体系。所有用户均从 `auth.html` 完成登录或注册，AIGC 独立注册入口已下线。

### 1.1 关键规则

- **租户严格映射企业**：一个企业 = 一个租户，不可一对多。
- **凭邀请码加入企业**：必须经过企业管理员二次审核。
- **个人注册**：默认进入个人空间，无需企业管理员审核。
- **新建企业/团队主体**：走入驻流程（`enterprise-onboarding.html`），由平台审核。

### 1.2 端到端角色

| 角色 | 职责 |
|------|------|
| 申请人 | 凭邀请码或自主注册申请加入企业 / 自助开通个人空间 |
| 企业管理员 | 生成与管理邀请码，二次审核入企申请，维护企业资料 |
| 平台运营 | 审核首次入驻、企业认证变更、入企申请 |

## 2. 核心概念

### 2.1 租户 = 企业

「租户」是一个抽象概念，平台中严格等同于「企业主体」：

- 1 个租户拥有 1 套企业资料（企业名称、营业执照、法人等）。
- 1 个租户下可以包含多个**工作空间**（如短剧组、市场组、IP 组）。
- 1 个用户只能归属一个企业租户（个人空间例外）。

### 2.2 三类入口

| 入口类型 | 适用场景 | 是否需要审核 |
|----------|----------|--------------|
| 凭邀请码加入企业 | 已存在企业的新成员 | 需企业管理员审核 |
| 创建企业/团队主体 | 新建企业租户 | 需平台审核（入驻审核） |
| 个人注册 | 自由职业者 / 学员 | 无需审核 |

### 2.3 邀请码

由企业管理员在管理员后台生成，状态机如下：

- **有效**：可正常使用
- **已过期**：超过有效期
- **已撤销**：管理员主动撤销
- **已用尽**：使用次数达到上限

## 3. 统一登录注册（auth.html）

> 入口：所有页面的「登录 / 注册」均跳转此页

### 3.1 页面结构

页面分左右两栏：

- **左侧**：品牌区，说明当前规则（租户=企业、入企需二审、个人注册免审核）
- **右侧**：登录 / 注册 双 Tab 切换

### 3.2 登录

支持两种登录方式：

| 方式 | 字段 | 说明 |
|------|------|------|
| 验证码登录 | 手机号 + 6 位验证码 | 演示验证码：`123456` |
| 密码登录 | 手机号 + 密码 | 支持「忘记密码」找回 |

登录前会校验账号状态：

- 账号审核中 → 提示「你的入企申请仍在审核中」
- 账号已驳回 → 提示「账号审核已驳回，请联系管理员」
- 账号正常 → 进入工作台

### 3.3 注册 · 三选一

#### 方式 A：凭邀请码加入企业

- 输入：姓名、手机号、密码、邀请码
- 邀请码会实时校验，校验通过会显示对应企业名（取自 `enterpriseProfile`）
- 提交后状态变为「审核中」，需要企业管理员通过后才能登录
- 邀请码可通过 URL 自动预填：`auth.html?invite=INV-XXXXXXXX`

#### 方式 B：创建企业/团队主体

- 点击「创建企业/团队主体」卡片
- 跳转 `enterprise-onboarding.html` 走完整入驻流程

#### 方式 C：个人注册

- 输入：姓名、手机号、密码
- 提交后立即可登录，自动进入个人空间

## 4. 企业管理员后台 · 企业资料

> 入口：`admin-console.html` → 顶部 Tab「🏛 企业资料」

「企业资料」是租户信息的唯一展示与变更入口，分为两个区域：

### 4.1 认证类信息（只读）

包含：企业全称、统一社会信用代码、企业类型、注册地址、法定代表人、法人证件号、营业执照文件。

变更入口：右上角「✏️ 申请变更」按钮 → 跳转 `enterprise-verification-prototype.html` 走重新审核。

> 注：所有租户级身份字段都走平台审核，避免企业身份被随意篡改。

### 4.2 运营类信息（可编辑）

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 对外简称 | 是 | 最多 20 字，保存后同步顶栏 banner 与页面标题 |
| 所属行业 | 否 | 文化传媒 / 影视制作 / 互联网 / 广告营销 / 教育培训 / 其他 |
| 管理员姓名 | 是 | 当前企业管理员姓名 |
| 管理员手机号 | 是 | 11 位手机号 |
| 管理员邮箱 | 是 | 标准邮箱格式 |
| 官方网站 | 否 | 可填 https:// 地址 |
| 对外联系人 / 电话 | 否 | 对外展示联系信息 |
| 企业简介 | 否 | 最多 500 字，带字符计数 |
| 备注 | 否 | 仅企业管理员可见，最多 200 字 |

按钮：

- **保存修改**：写入 `localStorage.enterpriseProfile`，立即生效
- **取消**：重新加载已保存的资料
- **重置**：恢复运营类字段默认值（认证类字段不变）

### 4.3 数据联动

保存后会同步刷新：

- 顶部 banner：`👑 企业管理员后台 · {对外简称}`
- 页面标题：`{对外简称} · 企业管理员后台`
- `account-basic-info-prototype.html` 的「企业信息」字段
- `auth.html` 邀请码注册时显示的企业名
- `enterprise-verification-prototype.html` 的企业全称、法人姓名（首次加载时同步）

## 5. 企业管理员后台 · 邀请管理

> 入口：`admin-console.html` → 顶部 Tab「📨 邀请管理」

### 5.1 列表视图

| 列 | 说明 |
|----|------|
| 邀请码 | 格式 `INV-XXXXXXXX`（8 位随机字符） |
| 邀请链接 | 点「复制链接」→ 复制 `auth.html?invite=邀请码` |
| 状态 | 有效 / 已过期 / 已撤销 / 已用尽 |
| 已用/上限 | 显示已使用次数与上限 |
| 有效期至 | 邀请码到期时间 |
| 创建人 | 默认 `cfeladmin` |
| 创建时间 | 邀请码生成时间 |
| 操作 | 仅「有效」状态可点「撤销」 |

### 5.2 筛选条件

- 状态：全部 / 有效 / 已过期 / 已撤销 / 已用尽
- 邀请码：模糊匹配
- 创建人：模糊匹配
- 「查询」/「重置」按钮

### 5.3 新建邀请

点击右上角「+ 新建邀请」打开弹窗：

| 字段 | 默认值 | 选项 |
|------|--------|------|
| 默认角色 | 发布员 | 发布员 / 制作主管 / 市场专员 / IP 主理人 / 财务 |
| 默认工作空间 | 短剧制作中心 | 短剧 / 市场 / IP 孵化组 |
| 可用次数 | 不限制 | 1 / 5 / 20 / 不限制 |
| 有效期 | 30 天 | 1 / 7 / 30 / 90 / 365 天 |

弹窗底部提示：「被邀请者注册后将提交入企申请，需在入驻审核中由企业管理员二次审核通过后才能登录。」

点「生成邀请」会：

- 生成新邀请码（如 `INV-XK9PZ2Q7`）
- 自动复制邀请链接
- 列表顶部新增一条记录

### 5.4 撤销

- 仅在「有效」状态下可撤销
- 撤销需二次确认
- 撤销后该邀请链接立即失效，但已通过该码提交的入企申请不受影响

## 6. 入驻审核 · 入企申请

> 入口：`admin-onboarding-review.html` → 工单类型 Tab「📨 入企申请」

### 6.1 工单类型

入驻审核现在支持三类工单：

| 工单类型 | 处理方 | 说明 |
|----------|--------|------|
| 入驻申请 | 平台运营 | 新企业 / 团队 / 个人首次入驻 |
| 企业认证变更 | 平台运营 | 已认证企业修改身份字段 |
| **入企申请** | **企业管理员** | **凭邀请码加入企业的新成员** |

### 6.2 入企申请工单卡片

工单卡片显示：

- 工单类型徽章：紫色「入企申请」
- 申请人姓名 + 申请加入的企业名
- 邀请码
- 申请人手机号 / 邮箱
- 风控标签：「已通过邀请码校验」「需校验申请人手机号 / 姓名」

### 6.3 审核操作

#### 通过

点「✅ 通过并加入企业」：

- `pendingEnterpriseMembers[].auditStatus` → `Approved`
- `authAccounts[].status` → `Active`（用户随后可登录）
- 自动写入 `demoMembers`（成员管理列表立刻可见，默认角色「发布员」、默认空间「短剧制作中心」）

#### 驳回

点「❌ 驳回」打开弹窗，选择驳回原因：

| 原因 | 适用场景 |
|------|----------|
| 申请人非本企业员工 | 邀请码外泄给无关人员 |
| 邀请码使用错误或来源不明 | 内部不知情的邀请码 |
| 手机号或姓名与企业花名册不一致 | 身份信息异常 |
| 申请角色与所属团队不匹配 | 邀请码角色配置错误 |
| 其他 | 自定义补充说明 |

驳回后：

- `pendingEnterpriseMembers[].auditStatus` → `Rejected`
- `authAccounts[].status` → `Rejected`
- 用户下次登录时会看到驳回提示

### 6.4 与首次入驻的区别

| 维度 | 首次入驻 | 入企申请 |
|------|----------|----------|
| 主体 | 新企业 / 团队 / 个人 | 已存在企业的新成员 |
| 审核方 | 平台运营 | 企业管理员 |
| 资质上传 | 必需 | 不需要 |
| 通过后效果 | 创建新租户 | 加入已有租户 |
| 驳回原因 | 材料不全等 | 身份不匹配等 |

## 7. 企业认证页双向同步

> 入口：`enterprise-verification-prototype.html`

企业认证页与「企业资料」Tab 共用 `enterpriseProfile` 数据：

### 7.1 加载时（认证页 → 企业资料）

打开认证页时会从 `enterpriseProfile` 读取并覆盖默认值：

- `enterpriseName` → 企业名称输入框
- `legalName` → 法人姓名输入框
- `certStatus` → 自动切换页面状态（已认证 / 审核中 / 已驳回 / 待提交）

### 7.2 保存时（企业资料 ← 认证页）

以下操作会触发回写：

- 修改企业名 / 法人名（输入框 blur 时）
- 点「提交认证」/「重新提交认证」→ `certStatus = pending`
- 点「撤回申请」→ `certStatus = draft`
- 演示状态切换（页面顶部 demo chips）→ 立即同步
- 演示通过审核 → `certStatus = verified`，并更新 `certUpdatedAt`

### 7.3 状态映射

| `enterpriseProfile.certStatus` | 认证页 `state.certStatus` | 中文标签 |
|--------------------------------|---------------------------|----------|
| `verified` | `VERIFIED` | 已认证 |
| `pending` | `PENDING` | 审核中 |
| `rejected` | `FAILED` | 已驳回 |
| `draft` | `UNVERIFIED` | 待提交 |

企业资料 Tab 顶部的「认证横幅」会用同一份状态字段显示徽章。

## 8. 端到端典型流程

### 流程 1：企业管理员邀请新成员

```mermaid
graph LR
A[企业管理员] -->|admin-console.html → 邀请管理| B[新建邀请]
B --> C[复制邀请链接]
C -->|微信/邮件发送| D[受邀人]
D -->|打开 auth.html?invite=...| E[填表注册]
E --> F[入企申请提交]
F -->|admin-onboarding-review.html| G[管理员二次审核]
G -->|通过| H[成员加入企业]
H --> I[受邀人登录工作台]
```

详细步骤：

1. 管理员在 `admin-console.html → 邀请管理` 点「+ 新建邀请」，配置角色与有效期。
2. 列表中复制邀请链接（`auth.html?invite=INV-XK9PZ2Q7`）。
3. 把链接发给受邀人。
4. 受邀人打开链接，邀请码自动预填，填写姓名/手机号/密码，提交。
5. 受邀人看到提示：「申请已提交，需企业管理员二次审核后才能登录」。
6. 管理员前往 `admin-onboarding-review.html`，工单类型选「📨 入企申请」。
7. 点「✅ 通过并加入企业」。
8. 受邀人在 `auth.html` 用手机号登录，进入工作台。
9. `admin-console.html → 成员管理` 看到新成员。

### 流程 2：管理员修改企业资料

1. `admin-console.html → 企业资料`。
2. 修改对外简称为「骋风天合·新形象」。
3. 点「保存修改」。
4. 顶部 banner 立即更新。
5. 任何打开着 `account-basic-info-prototype.html` 或 `auth.html` 的浏览器标签自动同步。
6. 下次有人凭邀请码注册，看到的企业名也会是新名字。

### 流程 3：企业认证字段变更

1. `admin-console.html → 企业资料 → ✏️ 申请变更`。
2. 跳转 `enterprise-verification-prototype.html`。
3. 修改企业名称 / 法人姓名等。
4. 点「提交认证」→ 状态变为「审核中」。
5. `enterpriseProfile.certStatus = pending` → 企业资料 Tab 顶部徽章变成「审核中」。
6. 平台运营在 `admin-onboarding-review.html` 工单类型「企业认证变更」中处理。
7. 通过后 `certStatus = verified`，企业资料中的认证字段被覆盖。

## 9. 状态机速查

### 9.1 入企申请状态机

| 状态 | 触发 | 用户能否登录 | 后续动作 |
|------|------|--------------|----------|
| `Pending` | 凭邀请码注册成功 | 否 | 等待企业管理员审核 |
| `Approved` | 管理员通过 | 是 | 写入成员管理 |
| `Rejected` | 管理员驳回 | 否 | 联系管理员重新邀请 |

### 9.2 邀请码状态机

| 状态 | 触发 | 是否可使用 |
|------|------|------------|
| `valid` | 新建邀请 | 是 |
| `expired` | 超过有效期 | 否 |
| `revoked` | 管理员撤销 | 否 |
| `usedup` | 已用 = 上限 | 否 |

### 9.3 企业认证状态机

| 状态 | 触发 | 可编辑 |
|------|------|--------|
| `verified` | 通过审核 | 否（需走变更） |
| `pending` | 提交审核 | 否 |
| `rejected` | 驳回 | 是（修改后重新提交） |
| `draft` | 撤回 / 初始 | 是 |

## 10. 常见问题

**Q1：邀请码已发出去，但企业资料里改了企业名，旧链接还能用吗？**  
A：能用。邀请码本身不带企业名，注册页显示的企业名实时从 `enterpriseProfile` 读取，因此始终是最新名字。

**Q2：管理员撤销了邀请码，已经用它提交了入企申请的人受影响吗？**  
A：不受影响。已提交的申请会留在「入企申请」工单队列，管理员仍可审批；撤销只影响后续新注册者。

**Q3：受邀人填错了手机号，怎么办？**  
A：管理员在「入企申请」工单中点驳回，选「手机号或姓名与企业花名册不一致」，受邀人需要重新走一次注册流程。

**Q4：个人注册的用户能不能后续加入企业？**  
A：可以。让企业管理员发邀请码，个人用户用同一手机号在 `auth.html?invite=...` 提交申请，审核通过即可加入企业（保留原个人账号）。

**Q5：邀请码可以指定加入某个工作空间和角色吗？**  
A：可以。新建邀请时配置「默认角色」「默认工作空间」，审核通过后这些字段会写入 `demoMembers`。

**Q6：管理员后台改企业资料需要平台审核吗？**  
A：分两类：
- **认证类**（企业名称、信用代码、法人、营业执照等）→ 必须走 `enterprise-verification-prototype.html` 重新审核
- **运营类**（简介、管理员手机邮箱、对外联系人等）→ 直接保存，无需审核

**Q7：邀请码的「有效期」可以无限长吗？**  
A：演示版本最长 365 天，正式版可配置；运营建议默认 30 天，避免长期外泄。

**Q8：一个手机号能注册多个企业账号吗？**  
A：不可以。统一身份入口下，1 个手机号对应 1 个账号。一个账号只能属于一个企业租户（个人空间例外）。
