## A. 治理类（管理后台 · 资金与预算 Tab）

### A1. 消费审核（事前审批） ✅ 原型已完成

- **优先级**：P0
- **状态**：**原型完成（2026-05-13）**。已在 admin-console.html 新增 Tab 4「🛡 消费审核」，包含阈值配置、待审批列表、审批历史、模拟触发四个完整流程，数据通过 localStorage 持久化。后续接入真实订单状态机时主要工作：
  - 在 demand-publish 提交时调用 `window.demoApp.evaluateSpendReview()` 判定是否触发
  - 触发后创建审批单（`addSpendApproval`），订单状态改为 PENDING_APPROVAL
  - 决策回写：approve 后走支付/锁资流程；reject 后订单退回草稿
  - 接入通知中心（C1）做事件推送
- **入口**：admin-console.html · Tab 4 「🛡 消费审核」（Tab 3 工具栏内有跳转入口）
- **背景**：当前成员发起订单后直接进入支付/锁资流程，企业管理员无法对大额或超预算订单做事前管控，资金安全风险高
- **范围**：
  - 设定审批阈值：单笔金额上限 / 团队预算占用比例上限 / 单成员月度累计上限
  - 命中阈值的订单进入"审批中"状态，不锁资、不扣款
  - 管理员审批通过 → 进入正常下单流程；驳回 → 订单退回草稿，附驳回理由
  - 审批超时（如 24h）自动催办或按规则默认拒绝
- **依赖**：
  - 角色权限：定义"谁有审批权"（一般是空间负责人 + 企业管理员）
  - 通知系统：审批待办需推送到管理员 + 申请人状态变更
  - 订单状态机：新增 "PENDING_APPROVAL" 状态
- **验收要点**：
  - 配置页可设置阈值规则（金额/预算占比/月度上限三选其一或组合）
  - 命中阈值订单不进入支付，状态显示"审批中"
  - 管理员侧有审批列表 + 批量审批
  - 审批日志可追溯（谁、何时、批准/驳回理由）
  - 移动端可审批（影响响应速度）

### A2. 充值审核

- **优先级**：P0
- **入口**：admin-console.html · Tab 3 工具栏 → 📥 充值审核
- **背景**：企业对公转账后，财务上传转账凭证，平台需审核到账才能把金额计入企业主账户。当前无审核入口
- **范围**：
  - 财务/出纳上传转账凭证（图片/PDF）+ 金额 + 流水号
  - 平台财务复核：核对银行流水 → 通过则金额入账、生成对应充值流水；驳回则原路退回（占位状态）
  - 多角色协作：财务发起 → 主管复核（视企业规模而定）
- **依赖**：
  - 文件上传服务
  - 银行流水接口（高级；初期可纯人工复核）
  - 审批流引擎（与 A1 共用）
- **验收要点**：
  - 申请单可看到凭证、金额、银行流水号
  - 通过/驳回均可备注
  - 通过后钱包对应账户余额实时增加，并写一条 type=recharge 的流水
  - 全流程时长统计（监控财务效率）

### A3. 开票汇总

- **优先级**：P1
- **入口**：admin-console.html · Tab 3 工具栏 → 🧾 开票汇总
- **背景**：现有钱包·发票管理只能看个人/当前空间的开票，企业管理员需要全企业级别的销售/采购发票汇总，用于报税、对账
- **范围**：
  - 销售发票汇总：按月/季度统计全企业开给客户的发票（金额、税率、状态）
  - 采购发票汇总：按月/季度统计全企业收到的供应商发票
  - 导出 Excel / 推送给财务系统
  - 异常发票预警（金额异常、抬头错误等）
- **依赖**：
  - 现有发票数据结构（钱包·发票管理）
  - 导出/打印基础设施
- **验收要点**：
  - 列表支持按时间、空间、状态多维筛选
  - 汇总数据准确（与单张发票之和一致）
  - Excel 导出格式与税务申报模板对齐

## B. 治理类（管理后台 · 其他 Tab）

### B1. 批量导入成员

- **优先级**：P1
- **入口**：admin-console.html · Tab 0 工具栏 → 📥 批量导入
- **背景**：企业上百人时单条邀请效率低
- **范围**：上传 CSV/Excel → 字段映射（姓名/邮箱/角色/空间）→ 预览校验 → 批量发起邀请
- **依赖**：邀请逻辑（已有）；模板文件下载；导入错误回显
- **验收要点**：错误行不阻塞其他成功导入；导入完成报告（成功/失败/原因）

### B2. 自定义角色 / 角色编辑

- **优先级**：P2
- **入口**：admin-console.html · Tab 2 工具栏 → + 新建角色；列表 → 编辑
- **背景**：现有 4 个预置角色（财务/制作主管/创作者/管理员）无法满足所有企业架构
- **范围**：
  - 编辑角色名、描述
  - 勾选权限点（与权限体系对齐）
  - 系统预置角色"只读"，自定义角色可编辑/删除
- **依赖**：权限点完整定义（PRD-1.2 已部分定义，需要补全）
- **验收要点**：删除角色时如有成员持有该角色需先转移；自定义角色不能拥有超管权限

### B3. 工作空间设置

- **优先级**：P2
- **入口**：admin-console.html · Tab 1 → 空间列表 → 设置按钮
- **范围**：
  - 重命名空间
  - 调整空间类型/绑定预算账户
  - 关闭/归档空间（需确保无进行中订单 + 余额为 0）
  - 成员加入/移出
- **依赖**：账户归档逻辑、订单状态检查
- **验收要点**：关闭空间不影响历史订单/流水查询

## C. 账号中心待补页面

### C1. 消息通知

- **优先级**：P2
- **入口**：account-center.html · 侧栏 → 🔔 消息通知
- **背景**：用户对哪些事件接收推送/邮件目前无法配置
- **范围**：
  - 按事件类型设置通知通道（站内 / 邮件 / 短信 / 钉钉/飞书）
  - 免打扰时段
  - 通知历史 + 已读状态
- **依赖**：通知中心后端（事件源、模板引擎、多通道发送）
- **验收要点**：偏好实时生效（5 分钟内）；关闭某事件后不再收到推送

### C2. 第三方账号

- **优先级**：P2
- **入口**：account-center.html · 侧栏 → 🔗 第三方账号
- **背景**：当前仅支持手机号/邮箱登录，企业用户希望接入飞书/微信单点登录
- **范围**：
  - 绑定/解绑：微信、飞书、钉钉、Google（按市场需求排序）
  - 绑定后支持单点登录
  - 解绑前检查：是否唯一登录方式
- **依赖**：各平台 OAuth 集成；账号合并/拆分逻辑
- **验收要点**：单点登录成功率 > 99%；解绑后该账号无法再登录

### C3. 隐私设置

- **优先级**：P3
- **入口**：account-center.html · 侧栏 → 🛡 隐私设置
- **背景**：合规要求（如《个人信息保护法》），用户需可控自己的数据可见性
- **范围**：
  - 个人主页对外可见字段勾选
  - 数据导出（《个人数据下载》合规要求）
  - Cookie 偏好
  - 已授权第三方应用列表 + 撤销
- **依赖**：数据导出服务；可见性元数据建模
- **验收要点**：导出包包含全部个人数据（订单、流水、设置）；撤销授权后第三方应用立即失效

### C4. 账号注销

- **优先级**：P3
- **入口**：account-center.html · 侧栏 → 🚪 账号注销 / 危险区按钮
- **背景**：合规要求用户有"被遗忘权"
- **范围**：
  - 注销前检查：余额清零、无进行中订单、无未结算流水
  - 二次确认 + 短信/邮件验证码
  - 7 天冷静期（期间可撤销）
  - 注销后数据保留期（按法规要求）
- **依赖**：资金清零规则、订单关闭规则、冷静期任务调度
- **验收要点**：注销前阻断条件清晰列出；冷静期内能找回；超期后用户名/邮箱可被其他人注册
