# 企业认证页 Prototype Brief（v0.2）

## 1. 页面名称
`账户中心 > 企业认证页`（[New]）

## 2. 页面目标
- 企业账号管理员在账户中心提交企业认证资料，并查看审核进度、认证结果、失败原因。
- 页面需要清晰区分 `未认证 / 审核中 / 已认证 / 认证失败` 四种状态，避免误操作和重复提交。
- 成功标志：资料可一次性完整提交；状态展示清晰；失败后能快速定位字段问题并重新提交；已认证后可发起变更流程。

## 3. 使用角色
- 企业账号管理员：可查看、提交、撤回申请、发起变更。
- 子账号：无查看权限，不可进入本页。
- 平台审核员：不在本页操作，审核发生在后台审核系统。

## 4. 页面入口 / 出口
- 入口：
  - `账户中心 > 企业认证`
  - [AI 推测 - 待确认] 某些企业能力开通前的引导入口
- 面包屑：
  - `首页 > 账户中心 > 企业认证`
- 出口：
  - 取消：返回账户中心首页或上一级设置页
  - 提交成功：停留当前页，状态切换为 `审核中`
  - 撤回成功：停留当前页，状态切换为 `未认证`
  - 变更申请成功：停留当前页，状态切换为 `审核中`

## 5. 页面信息架构
- 顶部标题区：页面标题、说明文案、认证状态标签
- 状态摘要区：状态说明、预计审核时效、关键提示文案
- 失败原因区：仅 `认证失败` 时展示失败摘要和字段级错误说明
- 认证信息表单区：
  - 企业信息
  - 法人信息
  - 对公账户信息
  - 补充信息
- 底部操作区：提交、重新提交、撤回申请、发起变更、取消

## 6. 核心模块说明
- 状态展示模块：统一展示当前认证状态、操作可用性和审核时效。
- 企业认证表单模块：收集企业名称、营业执照、法人身份信息、对公账户、企业官网。
- 附件上传模块：处理营业执照和身份证正反面上传、预览、替换、失败提示。
- 失败原因模块：消费后端返回的字段级结构化失败原因，并映射到页面级和字段级提示。
- 撤回 / 变更模块：在 `审核中` 支持撤回申请，在 `已认证` 支持发起变更企业认证信息。

## 7. 字段与数据展示规则

| 分组 | 字段 | 类型 | 必填 | 规则 | placeholder / 示例 | 错误提示 | 可编辑状态 |
|---|---|---|---|---|---|---|---|
| 企业信息 | 企业名称 | 文本输入 | 是 | 2-100 字；支持中英文、数字、括号、常见企业名称符号；自动 trim；需与营业执照主体一致 | 请输入营业执照上的企业全称 | `请输入企业名称` / `企业名称需为 2-100 字` | 未认证、认证失败、已认证发起变更时可编辑 |

| 补充信息 | 企业官网 | URL 输入 | 否 | 非必填；填写时需为合法 URL；建议自动补全 `https://`；最长 200 字 | 请输入企业官网，如 https://www.example.com | `请输入正确的网址` | 未认证、认证失败、已认证发起变更时可编辑 |

### 即时校验规则
- 文本类字段采用失焦校验。
- 银行账号输入过程中仅允许数字和空格，失焦后校验长度和格式。
- URL 在失焦后校验合法性。
- 上传完成后即时校验格式、大小、数量。
- 点击提交时进行全量校验，并滚动定位到首个错误字段。

## 8. 附件上传规则
- 上传控件统一使用卡片式上传区，支持点击上传。
- 首版支持 PC 端点击上传；拖拽上传 [AI 推测 - 待确认] 可作为增强能力，不作为必需。
- 上传中显示 loading 和进度状态。
- 上传成功后展示缩略图或文件名，并支持预览。
- 在可编辑状态下支持 `替换`、`删除`。
- 在只读状态下仅支持 `查看`。
- 不需要 OCR 自动识别营业执照或身份证信息。

### 单字段限制
- 营业执照：`1` 个文件，支持 `jpg/jpeg/png/pdf`，`<= 10MB`
- 法人身份证正面：`1` 个文件，支持 `jpg/jpeg/png`，`<= 10MB`
- 法人身份证反面：`1` 个文件，支持 `jpg/jpeg/png`，`<= 10MB`

### 页面级限制
- 附件总大小建议不超过 `30MB`
- 不支持多选后自动分配到多个字段

### 上传失败提示文案
- `上传失败，仅支持 jpg/jpeg/png/pdf`
- `上传失败，文件不能超过 10MB`
- `上传失败，请检查网络后重试`
- `图片不清晰或信息不完整，请重新上传`

## 9. 认证状态展示

### 状态枚举
- `未认证`
- `审核中`
- `已认证`
- `认证失败`

### 视觉语义
- 未认证：灰色标签
- 审核中：蓝色或橙色标签
- 已认证：绿色标签
- 认证失败：红色标签

### 状态文案
- 未认证：`您尚未完成企业认证，请填写资料并提交审核`
- 审核中：`认证资料已提交，平台将在五个工作日内完成审核`
- 已认证：`企业认证已通过，当前账号已完成企业主体认证`
- 认证失败：`本次认证未通过，请根据失败原因修改后重新提交`

### 展示位置
- 页面标题右侧展示状态标签
- 状态摘要区展示状态说明文案
- `审核中` 时展示固定时效：`预计五个工作日内完成审核`
- `认证失败` 时同时展示页面级失败摘要和字段级错误

## 10. 认证失败原因展示方式
- 失败原因由后端返回字段级结构化数据。
- 页面顶部展示红色 `Alert`：`认证失败，请根据以下原因修改后重新提交`
- 表单上方展示“失败原因卡片”，按条列出全部失败原因。
- 能映射到字段的失败原因，必须同步渲染到对应字段下方，并高亮字段边框。
- 无法映射到具体字段的系统级失败原因，显示在页面级失败卡片中。

### 建议返回结构
```json
{
  "status": "FAILED",
  "reasons": [
    {
      "field": "businessLicense",
      "code": "IMAGE_BLUR",
      "message": "营业执照信息模糊，无法识别，请重新上传清晰图片"
    },
    {
      "field": "bankAccountName",
      "code": "ACCOUNT_NAME_MISMATCH",
      "message": "开户户名需与企业主体一致"
    }
  ]
}
```

## 11. 审核中 / 已认证状态下页面是否可编辑

### 状态对应规则
- 未认证：可编辑，可提交。
- 认证失败：可编辑，回填上次提交内容，可重新提交。
- 审核中：不可编辑，但支持 `撤回申请`。
- 已认证：默认只读，但支持发起 `变更企业认证信息`。

### 页面表现建议
- 审核中：
  - 表单控件全部只读
  - 附件不可替换、不可删除，仅可查看
  - 主按钮显示为 `撤回申请`
  - 显示提示：`资料审核中，当前不可直接修改。如需调整，请先撤回申请。`
- 已认证：
  - 默认表单只读
  - 附件仅支持查看
  - 主按钮显示为 `变更企业认证信息`
  - 点击后进入“变更编辑态”，允许修改并重新提交，提交后状态切换为 `审核中`

## 12. 关键交互与状态

### 主操作按钮
- 未认证：`提交认证`、`取消`
- 认证失败：`重新提交认证`、`取消`
- 审核中：`撤回申请`、`返回`
- 已认证：`变更企业认证信息`、`返回`
- 已认证进入变更态后：`提交变更`、`取消`

### 提交流程
1. 用户填写信息并上传全部必填材料。
2. 点击 `提交认证` 或 `提交变更`。
3. 按钮进入 loading 态，防止重复点击。
4. 提交成功后停留当前页，刷新为 `审核中`。
5. 成功反馈：`提交成功，平台将在五个工作日内完成审核`

### 撤回流程
1. 用户在 `审核中` 点击 `撤回申请`。
2. 弹出二次确认弹窗。
3. 确认后撤回，页面刷新为 `未认证`，并回填最近一次提交内容。

### 撤回确认弹窗
- 标题：`确认撤回认证申请？`
- 正文：`撤回后当前审核将被终止，您可以重新修改资料并再次提交。`
- 确认按钮：`确认撤回`
- 取消按钮：`取消`

### 禁用条件
- 必填字段未填写完整
- 存在校验错误
- 附件仍在上传中
- 当前不在可编辑状态

## 13. 异常态 / 权限态 / 空态
- 加载中：表单区和状态区显示骨架屏
- 加载失败：显示错误态和 `重新加载` 按钮
- 提交失败：保留用户输入内容，显示全局错误提示；字段级错误直接映射到字段
- 无权限：子账号访问时展示无权限页，不展示认证详情
- 空态：`未认证` 即为空表单默认态，不单独使用空白页
- 附件加载失败：显示占位缩略图和 `重新加载`

## 14. 敏感字段展示规则
- 不提供敏感信息明文查看能力。
- 银行账号在只读场景下默认脱敏展示，例如 `**** **** **** 1234`。
- 身份证图片在只读状态下仅展示缩略预览，不提供放大后的明文细节强化展示。
- 不提供“查看原文”“显示全文”之类高敏入口。

## 15. 组件复用 / Design System
- 复用全局 `Form`、`Upload`、`Alert`、`Tag`、`Button`、`Modal`、`Skeleton` 组件。
- 失败原因卡片建议由 `Alert + List` 组合实现。
- 状态摘要区应复用账户中心统一状态样式，避免单页自定义风格。
- 若已有通用只读表单组件，优先用于 `审核中` 和 `已认证` 状态。

## 16. 给 AI coding 工具的实现提示
- 先做 PC 端页面原型，默认宽度 `1440px`。
- 首版只需保证 PC 端保真，小屏可仅保证基本浏览。
- 页面建议采用 `状态摘要 + 分组表单卡片 + 底部操作栏` 布局。
- 先不接真实接口，使用 mock 数据覆盖四种主状态，以及 `已认证进入变更态`。
- 建议将状态逻辑收敛为统一配置，例如：
  - `CERT_STATUS = UNVERIFIED | PENDING | VERIFIED | FAILED`
  - `pageMode = view | edit | change`
- 将字段配置、上传限制、状态可编辑性抽成常量，不要分散写死在组件内部。
- 失败原因消费逻辑建议支持：
  - 页面级 reason 列表
  - 字段级错误映射 `field -> message`
- 无权限态必须单独出页面，不要只隐藏内容。
- 不需要实现 OCR 或自动识别逻辑。
- mock 数据至少覆盖：
  - 极长企业名称
  - 官网为空
  - 上传失败
  - 多字段同时失败
  - 审核中只读 + 撤回
  - 已认证只读 + 进入变更态
- A11y 要求：
  - 按钮具备清晰可读的文本或 `aria-label`
  - 上传图片具备 `alt`
  - 表单控件必须有 `label`
  - 状态表达不能只依赖颜色
