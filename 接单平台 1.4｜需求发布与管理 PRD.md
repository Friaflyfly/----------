# 接单平台 1.4｜需求发布与管理 PRD

> 需求级别：**L3 平台型重需求**
> 裁决依据：涉及跨模块联动（钱包/入驻/订单/撮合）、状态机、核心业务对象新增（需求单/SPU/SKU）、权限模型变化、规则配置（类目驱动动态表单）、资金链路（锁资/扣款主体判定）

---

## 0. 术语约定（避免歧义）

平台中存在两个容易混淆的"空间"概念，本 PRD 与全平台 UI 一律按下列约定使用：

| 概念 | UI 文案 | 内部字段 / 标识 | 含义 |
|---|---|---|---|
| **容器层**（组织 / 团队 / 个人维度） | **团队**（含"个人") | `workspace` / `workspaceName` / `WORKSPACES` | 决定谁的预算、谁的成员、谁的权限 |
| **订单层**（单次交付的协同区） | **项目空间** / **项目协同空间** | `projectSpaceId` | 一个订单/已审核需求的聊天、文件、里程碑、交付物等协同载体 |

约定：
- UI 文案统一使用"团队"指代容器层；旧文案如"团队"、"团队范围"、"所属团队"、"个人"等已逐步替换为"团队"、"团队范围"、"所属团队"、"个人"。
- 数据字段 / API 仍保留 `workspace` 命名（避免连锁改动），仅 UI 层重命名。
- "空间"二字在用户可见的文案中**只用于"项目空间"** 这一含义。

## 1. 需求背景

当前平台已具备入驻准入（V1.3）、钱包结算（V1.2）等基础能力，但"需求方如何发布需求并管理已发布需求"的产品定义尚未完成，主要问题包括：

1. 平台同时存在**标品套餐**（按条/按篇计价的打包服务）和**自定义需求**（需求方自拟预算与规格），两种模式在产品层缺少统一发布框架，前后端无法共用同一套数据结构与提交流程。
2. 不同服务类目（视频制作、文案服务、图片设计、内容创作、资质服务）对应完全不同的表单字段、标签体系、服务等级和专属字段，但当前缺少类目驱动的动态表单方案，导致新增类目必须改代码。
3. 发布需求需要感知当前团队，不同空间对应不同扣款主体（个人账户 / 团队预算账户 / 企业主账户回退），这一规则在发布侧缺少产品化定义。
4. 需求发布后的状态流转（草稿 → 审核中 → 撮合中 → 待选承制方 → 进行中 → 已完成 / 已驳回）缺少统一状态机与逆向流转定义。
5. 平台套餐模式（PLATFORM_SKU）的服务范围、价格、交付规格已由平台目录锁定，**不需要人工审核**；但当前 PRD 未定义"自动审核通过"的状态机分支与审计标记，导致套餐流程与自定义流程在前端、审核后台、统计口径上无法区分对待。
6. 审核通过的需求（套餐自动通过 + 自定义人工通过）需要**自动创建项目协同空间**作为后续承制方报价、订单、交付协同的载体，但当前发布模块未定义对协同空间模块的调用契约与失败兜底。
7. 需求方缺少统一列表视图来管理不同空间、不同状态的需求。

## 2. 需求目标

### 2.1 本期目标

1. 建立"双模式发布"框架：平台套餐（SPU/SKU 选购）与自定义发布（类目 → 标签 → 等级 → 专属字段）统一在同一页面。
2. 建立类目驱动的动态表单引擎，不同一级类目渲染不同的二级类目、标签分组、服务等级、制作方式和专属字段，新增类目仅需扩展配置。
3. 建立需求状态机，覆盖从草稿到完成/驳回的完整生命周期，包含逆向流转与**套餐模式自动审核通过**分支定义。
4. 建立需求列表管理视图，支持多维筛选、状态聚合、操作快捷入口。
5. 建立发布侧的团队感知与扣款主体预判机制。
6. 建立**审核通过 → 自动创建项目协同空间**的对外调用契约与失败兜底链路（项目空间内部能力由协同模块独立迭代）。

### 2.2 本期不解决

1. 承制方报价与撮合引擎（属于撮合模块）。
2. 报价对比与选定承制方（属于报价模块）。
3. 订单创建与实际锁资扣款执行（属于订单 + 钱包模块联动）。
4. 项目协同空间的交付跟踪（属于协同模块）。
5. 平台运营审核后台的需求核定管理界面（属于运营后台模块）。
6. SPU/SKU 目录的运营管理后台。

## 5. 核心业务对象

### 5.1 需求单（Demand）

- 对象名称：需求单
- 对象定义：需求方发起的一次服务采购意图的结构化表达，包含类目、规格、预算、交期等信息
- 生命周期：草稿 → 审核中 / **自动通过** → 撮合中 → 待选承制方 → 进行中 → 已完成 / 已驳回 / 已取消
- 核心字段：见 §11.1
- 与其他对象关系：
  - 1 个需求单归属 1 个团队（workspace）
  - 1 个需求单对应 0~1 个订单（Order，下游模块创建）
  - 1 个需求单审核通过后对应 0~1 个项目协同空间（ProjectSpace，下游模块创建，详见 §5.6）
  - 套餐模式下 1 个需求单引用 1 个 SKU 快照（snapshot）
  - 自定义模式下 1 个需求单引用 1 个类目配置（CategoryConfig）

### 5.2 SPU（Standard Product Unit）

- 对象名称：标准产品单元
- 对象定义：平台标品服务的服务类型级抽象，如"营销短视频制作包"
- 生命周期：创建 → 上架 → 下架（运营管理，本期不涉及后台）
- 核心字段：`spu_id`, `name`, `icon`, `desc`, `upcoming`（是否即将上线）, `skus[]`
- 与其他对象关系：1 SPU 包含多个 SKU

### 5.3 SKU（Stock Keeping Unit / 档位）

- 对象名称：库存保管单元 / 套餐档位
- 对象定义：SPU 下的具体规格档位，锁定数量、价格、交付规格与条款版本
- 生命周期：创建 → 上架 → 调价/调量 → 下架
- 核心字段：`sku_id`, `display_name`, `quantity{value, unit}`, `price{amount, currency}`, `service_scope{highlights[], detail_html}`, `delivery{lead_time_days, full_delivery_days, delivery_notes}`, `commercial{tax_included, price_note}`, `display{status, sort_order, badge}`, `terms_version`, `sku_code`, `updated_at`
- 与其他对象关系：被需求单以**快照形式**（深拷贝）引用，快照一旦写入需求单即不再随目录更新变化

### 5.4 类目配置（CategoryConfig）

- 对象名称：类目配置树
- 对象定义：驱动自定义发布表单动态渲染的配置数据，按一级类目组织
- 生命周期：由运营配置，前端消费
- 核心字段：`l1_id`, `name`, `icon`, `desc`, `sub{l2_id: name}`, `tags{groupName: string[]}`, `levels[{id, name, desc}]`, `methods[]`, `dyn[{id, label, type, placeholder, options, required}]`
- 与其他对象关系：自定义模式需求单引用类目配置中的字段定义

### 5.5 草稿（Draft）

- 对象名称：发布草稿
- 对象定义：客户端暂存的未提交需求表单状态
- 生命周期：创建 → 恢复 → 提交后清除 / 手动重置后清除
- 存储位置：客户端 localStorage（一期）
- 与其他对象关系：恢复后填充到发布页表单，提交后转化为需求单

### 5.6 项目协同空间（ProjectSpace · 外部对象）

- 对象名称：项目协同空间
- 对象定义：承载需求审核通过后报价、订单、交付协同的工作载体；由协同模块定义与维护，本模块仅作为**调用方**触发其创建
- 生命周期：由协同模块管理（创建 → 进行中 → 归档 / 关闭）
- 核心字段（本模块仅关注以下字段，完整模型见协同模块 PRD）：
  - `projectSpaceId`：项目空间唯一标识（由协同模块返回）
  - `demandId`：归属需求单 ID（本模块写入）
  - `workspace`：归属团队 ID（沿用需求单的团队）
  - `createdFrom`：创建来源（`AUTO_APPROVED` / `MANUAL_APPROVED`，用于审计）
- 与其他对象关系：
  - 1 个项目空间对应 1 个需求单（1:1）
  - 创建时机：需求审核通过即创建（套餐模式系统自动通过 = 提交时；自定义模式 = 审核员通过时）
  - 与订单的关系由协同模块定义，本模块不感知

## 6. 对象关系说明

```
团队（Workspace）
  └── 1:N ── 需求单（Demand）
                ├── type=PLATFORM_SKU ── 引用 SKU 快照（Snapshot）
                │                            └── 来源于 SPU → SKU 目录
                ├── type=CUSTOM ── 引用类目配置（CategoryConfig）
                │                      ├── 一级类目 → 二级类目
                │                      ├── 标签分组
                │                      ├── 服务等级
                │                      ├── 制作方式
                │                      └── 专属字段定义
                └── 审核通过后 1:1 ── 项目协同空间（ProjectSpace · 外部模块）
                                          ├── 套餐模式：提交即创建（AUTO_APPROVED）
                                          └── 自定义模式：审核员通过时创建（MANUAL_APPROVED）
```

- 需求单与团队为 N:1 关系，通过 `workspace` 字段关联
- 需求单与 SKU 快照为 1:1 内嵌关系（仅套餐模式），快照为深拷贝不随原 SKU 变化
- 需求单与类目配置为引用关系（仅自定义模式），通过 `l1`/`l2` 关联
- 需求单与项目协同空间为 1:1 关系，通过冗余字段 `projectSpaceId` 关联；项目空间数据由协同模块持有，本模块只读引用

### 6.1 生命周期联动

- 团队**冻结**时：该团队下的需求单状态不变，列表页继续展示（只读），不允许在该团队下新增需求；冻结期满或空间恢复后操作能力自动恢复
- 团队**删除**时：该团队下的需求单逻辑保留（数据不物理删除），列表展示依赖冗余字段 `workspaceName`；不允许在已删除空间下新增需求
- SPU 下架时：已引用该 SPU SKU 的历史需求单不受影响（快照已锁定），新发布不再可选
- 类目配置变更时：已提交的需求单不受影响（已落库），仅影响新发布
- 需求单删除方式：**逻辑删除，不设 TTL**（一期不提供前端删除入口）；超过 2 年的 `completed` 状态需求可由后端运维迁移到冷存归档，但不物理删除
- **项目协同空间联动**：
  - 套餐模式需求单提交成功 → 同步触发项目空间创建（含失败兜底，详见 §17.9）
  - 自定义模式需求单审核通过 → 同步触发项目空间创建
  - 需求单 `cancelled`（撤回时尚未审核通过）→ **不创建**项目空间
  - 需求单 `rejected` → 不创建项目空间；重提通过后再创建
  - 项目空间创建失败时，需求单仍保持已审核通过状态（不回滚），通过补偿机制重试创建
- 不允许出现"无团队归属的需求单"
- 不允许出现"已审核通过但无项目空间"的需求单超过 24 小时（由补偿任务保障最终一致）

## 7. 总体流程

### 7.1 主流程 · 平台套餐模式（自动审核通过）

1. 需求方进入发布页，默认选中"选平台套餐"模式
2. 浏览 SPU 列表，选择可用的服务类型
3. 在该 SPU 下选择具体档位（SKU）
4. 系统自动锁定总价、交付周期，生成 SKU 快照
5. 填写补充信息（Brief）：标题、详细描述、交期说明、参考资料
6. 确认实时摘要，点击提交
7. 系统校验必填项 → 校验入驻资料完善状态 → 生成需求 ID → 写入需求数据
8. **系统判定 `requirementType = PLATFORM_SKU` → 直接置状态为 `auto-approved`**（业务等价于 `matching`，详见 §8）
9. **系统同步触发项目协同空间创建** → 写回 `projectSpaceId`（失败兜底详见 §17.9）
10. 同步派发撮合事件到撮合引擎（不进入运营审核队列），清除草稿
11. 跳转至需求列表页，Toast "需求已自动通过审核 · 项目空间已创建"

> **设计要点**：套餐模式的服务范围、价格、交付规格、合同条款均由 SKU 快照锁定，平台已审核标品目录本身，单次需求无需重复人工审核。

### 7.2 主流程 · 自定义发布模式（人工审核）

1. 需求方切换到"自定义发布"模式
2. Step 1：选择一级类目 → 二级类目
3. Step 2：在 5 个标签分组中多选标签
4. Step 3：选择服务等级（单选）+ 制作方式（多选）
5. Step 4：填写类目专属字段（动态表单）
6. Step 5：填写标题、描述、预算范围、交期、参考资料
7. 确认实时摘要，点击提交
8. 系统校验 → 入驻拦截 → 生成需求 ID → 写入需求数据
9. 状态置为 `auditing`，进入运营审核队列，清除草稿
10. 跳转至需求列表页

### 7.3 后台协同流程（本期仅定义接口，后台 UI 属于运营后台模块）

1. 运营审核员在审核后台查看 `auditing` 状态的需求（**不含套餐模式需求**，套餐模式不入审核队列）
2. 审核员核定需求完整度与预算合理性
3. 审核通过 → 状态变为 `matching`，**系统同步触发项目协同空间创建**，进入撮合引擎
4. 审核驳回 → 状态变为 `rejected`，记录驳回原因（不创建项目空间）
5. 撮合完成 → 状态变为 `quoting`
6. 需求方选定承制方 → 状态变为 `in-progress`
7. 验收结算完成 → 状态变为 `completed`

### 7.4 上下游协同流程

1. 需求提交后：
   - 套餐模式 → 写入需求数据（本模块）→ **不通知**审核模块 → **同步调用协同空间创建**接口 → 派发撮合事件
   - 自定义模式 → 写入需求数据 → 通知运营审核模块（下游）
2. 自定义需求审核通过后 → **同步调用协同空间创建**接口 → 通知撮合引擎（下游）→ 撮合引擎回写 `quotesCount`
3. 项目协同空间创建失败时 → MQ 异步重试 + 死信队列兜底（详见 §17.9），不阻塞主流程
4. 选定承制方后 → 订单模块创建订单 → 钱包模块执行锁资
5. 锁资失败时 → 钱包页可深链跳回发布页（`?from=wallet&demandId=xxx`）调整金额

## 8. 状态机

### 8.1 状态定义

| 状态 | 含义 | 适用模式 | 允许操作 | 下一状态 |
|---|---|---|---|---|
| draft | 草稿（客户端暂存，不入库） | 套餐 / 自定义 | 编辑 / 提交 / 重置 | auditing / auto-approved |
| auditing | 审核中 · 平台运营核定需求 | 仅自定义 | 查看详情 / **撤回** | matching / rejected / cancelled |
| **auto-approved** | **自动审核通过 · 系统判定 · 业务等价于 matching** | **仅套餐** | **查看详情 / 查看项目空间** | **quoting** |
| matching | 撮合中 · 承制方报价中（人工审核通过后） | 仅自定义 | 查看详情 / 查看项目空间 | quoting |
| quoting | 待选承制方 · 已收到报价 | 套餐 / 自定义 | 对比报价 / 选定 | in-progress |
| in-progress | 进行中 · 承制方制作中 | 套餐 / 自定义 | 查看项目进度 | completed |
| completed | 已完成 · 已结算 | 套餐 / 自定义 | 查看归档 | — |
| rejected | 已驳回 · 未通过人工核定 | 仅自定义 | 修改并重提 | auditing |
| cancelled | 已取消 · 需求方主动放弃 | 套餐 / 自定义 | 查看归档 | — |

> **`auto-approved` 设计说明**：
> - 该状态作为状态机层面的**审计标记**，明确区分"系统自动通过"与"人工审核通过"，便于审核 SLA 统计、套餐转化率分析、合规追溯
> - 业务行为上等价于 `matching`：均触发撮合引擎、均创建项目协同空间、均允许查看项目空间入口
> - 列表页状态聚合卡片中，`auto-approved` 与 `matching` **合并展示为"撮合中"**，对需求方视角无差异（详见 §10.2）
> - 套餐模式不会出现 `auditing` 状态；自定义模式不会出现 `auto-approved` 状态

### 8.2 状态迁移规则

| 迁移 | 迁移条件 | 触发方 | 系统动作 | 是否可逆 | 影响其他模块 |
|---|---|---|---|---|---|
| draft → auditing | 自定义模式 · 提交校验通过 + 入驻资料完善 | 需求方 | 生成需求 ID，写入库，清除草稿 | 否（无逆向回 draft） | 通知审核模块 |
| **draft → auto-approved** | **套餐模式 · 提交校验通过 + 入驻资料完善** | **需求方（系统判定）** | **生成需求 ID，写入库，清除草稿，写入 `auditMode=auto` + `auditedAt=now` + `auditedBy=system-auto`** | **否（无逆向回 draft）** | **同步调用项目协同空间创建 + 派发撮合事件，不通知审核模块** |
| auditing → matching | 运营审核通过 | 审核员 | 更新状态，写入 `auditMode=manual` + `auditedAt=now` + `auditedBy=审核员 ID` | 否 | **同步调用项目协同空间创建** + 触发撮合引擎 |
| auditing → rejected | 运营审核驳回 | 审核员 | 更新状态，记录驳回原因 | — | 通知需求方，不创建项目空间 |
| auditing → cancelled | 需求方主动撤回 | 需求方 | 更新状态 + 记录撤回时间，**通知审核队列移除**该需求 | 否（不可恢复，需重新发布） | 通知审核模块移除队列，不创建项目空间 |
| rejected → auditing | 需求方修改后重提 | 需求方 | 更新需求数据，状态回 auditing | — | 重新进入审核队列 |
| matching → quoting | 收到承制方报价（≥1 家） | 撮合引擎 | 更新 quotesCount | 否 | — |
| **auto-approved → quoting** | **套餐模式 · 收到承制方报价（≥1 家）** | **撮合引擎** | **更新 quotesCount** | **否** | **—** |
| quoting → in-progress | 需求方选定承制方 | 需求方 | — | 否 | 订单模块创建订单，钱包模块锁资 |
| in-progress → completed | 验收结算完成 | 系统 | — | 否 | 钱包模块结算 |

### 8.3 逆向流转定义

- `auditing` 状态：**支持需求方主动撤回**，撤回后状态变为 `cancelled`（不回退到 `draft`，不删除需求记录）。撤回操作通知审核队列移除该需求。撤回后如需重发，走"新建需求"流程，可基于历史需求快速复制（二期能力）。
- `auto-approved` 状态：**一期不支持撤回**（与 `matching` 同口径）。理由：套餐模式自动通过即创建项目空间 + 派发撮合事件，撤回需联动项目空间归档 + 通知撮合引擎 + 通知已报价承制方，复杂度等同于二期"关闭需求"流程。一期套餐需求若需取消，按客服工单走人工兜底。
- `rejected` 状态：允许修改并重提，重提后回到 `auditing`，原驳回记录保留。
- `matching` / `quoting` 状态：一期**不支持**取消已撮合的需求。二期视业务诉求引入"关闭需求"流程（区别于"撤回"，需配合：① 原因说明 ② 通知所有已报价承制方 ③ 若已锁资需联动钱包模块解锁 ④ **联动协同模块归档项目空间**）。
- `in-progress` 状态：涉及订单取消与资金退回，属于订单/钱包模块逆向流程，本模块不处理。
- 一期不涉及冲抵/冲正单据。`cancelled` 状态仅由 `auditing → cancelled` 一条路径产生，无资金联动、无项目空间联动。后续若新增 `auto-approved → cancelled` / `matching → cancelled` / `quoting → cancelled` / `in-progress → cancelled` 路径，需配合关联模块（撮合 / 协同 / 钱包）的补偿机制实现。

### 8.4 触达规则

| 状态变化 | 通知对象 | 通知时机 | 一期实现 | 二期实现 |
|---|---|---|---|---|
| draft → auditing | 运营审核员 | 立即 | 消息队列异步推送至审核队列（重试 3 次：5s / 30s / 120s 指数退避，超限写入 DLQ + 告警运营群） | 同一期 |
| **draft → auto-approved** | **需求方** | **立即** | **同步：提交接口返回 200 + Toast "需求已自动通过审核 · 项目空间已创建" + 跳转列表页**（无需通知审核员，不进入审核队列） | **同一期** |
| **draft → auto-approved** | **协同模块** | **立即** | **同步调用 `POST /api/project-spaces` 创建项目空间，失败走 §17.9 兜底** | **同一期** |
| auditing → cancelled | 运营审核员 | 立即 | 消息队列推送"撤回事件"，审核队列移除该需求 | 同一期 |
| auditing → rejected | 需求方 | 立即 | Toast + 列表页可见 rejected 状态 | 补站内消息 + 邮件 |
| auditing → matching | 需求方 | 立即 | 列表页可见 matching 状态（依赖手动刷新） | 补站内消息 |
| **auditing → matching** | **协同模块** | **立即** | **同步调用 `POST /api/project-spaces` 创建项目空间，失败走 §17.9 兜底** | **同一期** |
| matching / auto-approved → quoting | 需求方 | 收到首条报价时 | 列表页可见 quotesCount + `?highlight=` 高亮入口 | 补站内消息 + 邮件 |

> **通知策略口径**：一期通知方式以 Toast + 页面跳转 + 列表展示为主，**不做站内信/短信/邮件**，站内消息系统作为独立基建在二期统一落地，不为本模块单独开发。
> 审核通知失败兜底：异步重试 3 次失败后写入死信队列（DLQ），同时推送告警到运营群，由人工补处理。
> **项目空间创建失败兜底**：同步调用失败时不阻塞主流程，由后端补偿任务异步重试（重试 3 次：5s / 30s / 120s 指数退避，超限写入 DLQ + 告警），详见 §17.9。

## 9. 功能模块设计

### 9.1 发布模式选择模块

- 模块目标：让需求方在进入发布流程时明确选择套餐购买或自定义发布
- 服务对象：需求方
- 核心能力：两张模式卡片切换，切换时清空对方模式数据并联动区块显隐
- 入口：发布需求页顶部
- 出口：决定后续展示哪套表单流程
- 与其他模块关系：控制"套餐选购模块"和"类目表单模块"的显隐

### 9.2 套餐选购模块（SPU / SKU）

- 模块目标：让需求方通过标品目录选择服务套餐，锁定价格与交付规格
- 服务对象：需求方
- 核心能力：SPU 网格展示 → SKU 档位选择 → 价格/交期/条款锁定 → 快照生成
- 入口：选中"选平台套餐"模式后展示
- 出口：选定 SKU 后自动将价格/交期写入基本信息区
- 与其他模块关系：写入的 SKU 快照附在需求单数据上

### 9.3 类目驱动表单模块（自定义模式 Step 1-4）

- 模块目标：按类目配置动态渲染表单，收集类目相关的结构化信息
- 服务对象：需求方
- 核心能力：
  - Step 1：一级类目卡片 → 二级类目药丸
  - Step 2：5 组标签多选
  - Step 3：服务等级单选 + 制作方式多选
  - Step 4：专属字段动态表单
- 入口：选中"自定义发布"模式后展示
- 出口：收集的数据写入需求单 `tags`, `level`, `methods`, `dyn` 字段
- 与其他模块关系：类目配置数据从配置 API 获取

### 9.4 基本信息模块（Step 5 / 通用）

- 模块目标：收集所有需求共通的基础信息
- 服务对象：需求方
- 核心能力：标题/描述/预算/交期/参考资料输入
- 入口：始终展示，两种模式共享
- 出口：写入需求单核心字段
- 与其他模块关系：套餐模式下预算区变为只读锁定区

### 9.5 实时摘要模块

- 模块目标：帮助需求方在填写过程中实时确认信息完整度
- 服务对象：需求方
- 核心能力：右侧 sticky 面板，汇总已填信息 + 5 步进度条 + 扣款主体预判
- 入口：始终展示在页面右侧
- 出口：纯展示，不产生数据
- 与其他模块关系：消费所有表单模块的实时状态

### 9.6 需求列表模块

- 模块目标：让需求方集中管理所有已发布的需求
- 服务对象：需求方
- 核心能力：状态聚合卡片 + 多维筛选 + 列表展示 + 状态感知操作按钮
- 入口：侧边栏"我的需求"
- 出口：跳转至报价对比页/项目空间/发布页（按状态不同）
- 与其他模块关系：读取需求数据，不修改

## 10. 页面 / 后台详细设计

### 10.1 发布需求页（demand-publish.html）

**页面目标**：需求方一站式完成需求信息填写并提交

**页面结构**：
- 页头：标题 + 当前团队标签 + 说明文案
- 左列（主区）：
  - Step 0：发布方式选择（两张模式卡片）
  - 套餐流程区（仅套餐模式可见）：SPU 网格 → SKU 网格
  - Step 1：类目选择（仅自定义模式可见）
  - Step 2：标签体系（仅自定义模式可见）
  - Step 3：服务等级 + 制作方式（仅自定义模式可见）
  - Step 4：类目专属字段（仅自定义模式可见）
  - Step 5/Brief：基本信息 + 预算交期（始终可见，内容随模式变化）
- 右列（摘要区）：实时摘要 sticky 面板
- 底部（提交栏）：审核提示 + 暂存草稿 + 重置 + 提交

**关键操作**：
| 操作 | 触发条件 | 动作 | 反馈 |
|---|---|---|---|
| 切换发布模式 | 点击模式卡片 | 清空对方模式数据，切换区块显隐 | Toast 提示 |
| 选择 SPU | 点击 SPU 卡片 | 展开 SKU 网格 | Toast 提示 |
| 选择 SKU | 点击 SKU 卡片 | 写入价格/交期/快照，更新摘要 | Toast 提示 + 摘要更新 |
| 选择一级类目 | 点击类目卡片 | 清空标签/等级/方式/专属字段，重新渲染 | Toast + 闪光提示 |
| 暂存草稿 | 点击"暂存草稿"按钮 | 序列化表单状态到 localStorage | Toast "草稿已暂存" |
| 重置 | 点击"重置"按钮 | 二次确认后清空所有数据 + 清除草稿 | Toast "已重置" |
| 提交（套餐模式） | 点击"提交需求"按钮 | 校验 → 入驻拦截 → 生成需求 → **自动审核通过** → 触发项目空间创建 → 跳转列表页 | Toast "需求已自动通过审核 · 项目空间已创建" + 3s 后跳转 |
| 提交（自定义模式） | 点击"提交需求"按钮 | 校验 → 入驻拦截 → 生成需求 → 入审核队列 → 跳转列表页 | Toast "已提交，等待平台核定" + 3s 后跳转 |

**空态 / 失败态 / 权限态**：
- 未选类目时：标签/等级/专属字段区显示"请先选择一级类目"占位文案
- 未选 SPU 时：SKU 区隐藏
- 企业管理员视角：显示黄色阻断横幅 + 提交/暂存按钮禁用
- 未配预算空间：正常发布但摘要中显示"⚠️ 团队未配预算 → 回退企业主账户"
- 入驻未完善：提交时弹出入驻资料完善拦截弹窗

**Design System / 组件依赖**：
- 步骤卡片（`.step-card`）：含步骤编号（完成/进行中/未到）+ 标题 + 提示
- 模式卡片（`.mode-card`）：双卡横排，active 态带蓝色渐变
- SPU/SKU 卡片：网格布局，SKU 含数量/价格/服务范围/交付周期
- 类目卡片（`.l1-card`）：5 列网格，含图标/名称/简述
- 标签药丸（`.l2-pill`, `.chip`）：支持单选/多选
- 服务等级卡片（`.level-card`）：4 列网格
- 动态表单（`.dyn-form`）：2 列网格，支持 text/number/select
- 实时摘要面板（`.summary`）：sticky 定位，含进度条
- 提交栏（`.submit-bar`）：sticky 底部，含审核提示横幅

### 10.2 需求列表页（demand-list.html）

**页面目标**：需求方查看和管理所有团队下的需求与状态

**页面结构**：
- 页头：标题 + "发布新需求"按钮
- 状态聚合卡片栏：5 张卡片横排（全部 / 审核中 / 撮合中 / 待选承制方 / 进行中）
  - **"撮合中"卡片合并展示 `matching` + `auto-approved`** 两类需求的计数；需求行的状态徽章按真实状态展示（自定义=橙→蓝转换；套餐=直接蓝），但聚合统计口径不区分
  - `rejected`、`cancelled`、`completed` 状态**不单独占聚合卡片位**，仅可通过类目下拉旁的"状态"二级筛选进入；这三类状态的计数在卡片栏右上角以"已驳回 N · 已取消 N · 已完成 N"灰色小字辅助展示
- 工具栏：团队范围 + 类目筛选 + 搜索 + 结果计数
- 列表区：7 列网格表头 + 数据行

**关键操作**：
| 操作 | 触发条件 | 动作 | 反馈 |
|---|---|---|---|
| 点击状态卡片 | 点击任意聚合卡片 | 按该状态筛选列表 | 卡片高亮 + 列表刷新 |
| 切换团队范围 | 下拉选择 | 按当前团队/所有团队过滤 | 列表刷新 + 计数更新 |
| 搜索 | 输入关键词 | 按需求名/需求号模糊匹配 | 列表实时过滤 |
| 对比报价（quoting） | 点击红色按钮 | 跳转报价对比页 | — |
| 查看进度（in-progress） | 点击蓝色按钮 | 跳转项目协同空间 | — |
| 修改并重提（rejected） | 点击按钮 | 跳转发布页 | — |

**空态**：
- 无符合条件的需求时显示空状态："📭 没有符合条件的需求" + "→ 立即发布一个新需求"链接

**Design System / 组件依赖**：
- 状态聚合卡片（`.stat-card`）：含标签 + 数值 + 后缀，active 态高亮
- 列表行（`.list-row`）：7 列网格，hover 态变色
- 状态徽章（`.status-badge`）：按状态配色（审核中=橙 / 撮合中=蓝 / **自动通过=蓝带 ✓ 标识** / 待选=红 / 进行中=绿 / 已驳回=深红 / 已取消=灰 / 已完成=灰）
  - `auto-approved` 状态文案展示为"撮合中"，但右上角带 ✓ 微标识表示"自动通过"，悬浮 tooltip "套餐模式 · 系统自动审核通过"
- 操作按钮（`.btn-link`）：小尺寸，按状态区分样式（primary/urgent/ghost）
- 项目空间入口（`.btn-project-space`）：状态为 `auto-approved` / `matching` / `quoting` / `in-progress` 时显示"进入项目空间"链接（`projectSpaceId` 存在时可点击；缺失时灰显 + tooltip "项目空间创建中..."）

### 10.3 查询性能约束

- 需求列表默认带团队范围过滤（"所有团队"仍限于用户参与的空间），不允许无条件全量查询
- 支持精确搜索字段：需求 ID
- 支持模糊搜索字段：需求标题
- 排序字段：创建时间倒序
- **分页策略**：
  - **后端 API 一期即支持** `page` + `pageSize` 参数（默认 `pageSize=50`），保证后续启用分页无需接口改造
  - **前端一期不传分页参数**，默认全量渲染（预估单用户需求量 <100，DOM <2000 节点性能可接受）
  - **二期启用分页阈值**：当统计数据中"单用户需求均值 >100"或"P95 >200"时，前端切换为分页查询；同时引入虚拟滚动作为兜底
- 导出能力：一期不做

| **auditMode** | **审核模式** | **enum** | **系统写入** | **—** | **否** | **`auto`（套餐模式）/ `manual`（自定义模式）；提交时由 `requirementType` 决定，不可后续变更** |
| **auditedAt** | **审核完成时间** | **datetime** | **系统写入** | **null** | **否** | **套餐：提交即写入；自定义：审核员通过/驳回时写入；驳回后重提清空再重写** |
| **auditedBy** | **审核执行人 ID** | **string** | **系统写入** | **null** | **否** | **套餐：固定值 `system-auto`；自定义：审核员账号 ID** |
| **rejectReason** | **驳回原因** | **string** | **审核员写入** | **null** | **否** | **仅 status=rejected 时有值；重提后保留历史，不清空** |
| **projectSpaceId** | **项目协同空间 ID** | **string** | **系统写入** | **null** | **否** | **审核通过后由协同模块返回；创建中或失败时为 null（详见 §17.9 兜底）** |
| **projectSpaceCreatedAt** | **项目空间创建时间** | **datetime** | **系统写入** | **null** | **否** | **创建成功时写入，用于 SLA 监控** |
| **cancelledAt** | **撤回时间** | **datetime** | **系统写入** | **null** | **否** | **仅 status=cancelled 时有值** |

### 11.2 敏感字段 / PII 规则

本模块**不直接收集**手机号、身份证号、银行卡号等 PII 字段。涉及的金额类字段（预算/套餐价格）为业务公开信息，不需要脱敏。

需求描述（`desc`）虽然可能包含用户自行输入的敏感信息（如联系方式、微信号），**一期不做自动内容扫描**，依赖平台审核员人工核定时识别并要求修改。二期视审核反馈数据决定是否补充自动敏感词扫描与提醒机制。

## 12. 业务规则与配置规则

### 12.1 核心业务规则

**类目驱动规则**：
- 触发条件：选中一级类目
- 判断条件：类目配置树中是否存在该 l1_id
- 系统动作：替换二级类目列表、标签分组、服务等级、制作方式、专属字段
- 影响范围：仅影响当前表单实例，不影响历史需求
- 切换一级类目时，所有下级已选数据清空

**SKU 快照规则**：
- 触发条件：选中 SKU
- 系统动作：深拷贝 SKU 完整数据 + 冗余 spu_id/spu_name
- 快照一旦写入需求单即锁定，不随后续 SKU 调价/下架变化
- `terms_version` 字段用于后续条款追溯

**团队扣款主体判定规则**：
| 空间类型 | 扣款主体 | 补充提示 |
|---|---|---|
| 个人 | 个人账户 | — |
| 团队（已配预算） | 本团队预算账户 | — |
| 团队（未配预算） | 企业主账户（回退） | 摘要 + Toast 提示 |
| 企业管理员视角 | — | 阻断发布 |

**提交校验规则**：

自定义模式校验优先级（全部通过方可提交）：
1. 类目已选（l1 + l2）
2. 至少 1 个标签
3. 服务等级已选 + 至少 1 种制作方式
4. 类目专属必填字段全部非空
5. 标题 + 描述 + 预算范围 + 交期 全部非空
6. budgetMin ≤ budgetMax

套餐模式校验优先级：
1. 已选 SPU + SKU（skuSnapshot 存在）
2. 标题 + 描述 + 交期说明 全部非空

**入驻资料拦截规则**：
- 触发条件：点击"提交需求"时 `profileComplete = false`
- 系统动作：弹出入驻资料完善弹窗（含"立即完善"跳转入驻页 / "Demo 跳过"快捷标记）
- 拦截后不执行提交

**套餐模式自动审核通过规则**：
- 触发条件：提交时 `requirementType = PLATFORM_SKU`
- 判断条件：
  - 必填项校验通过（参见上文"提交校验规则"套餐模式分支）
  - SKU 快照存在且 `terms_version` 与目录一致
  - 入驻资料完善
- 系统动作：
  1. 写入需求数据，状态直接置为 `auto-approved`（不进入 `auditing`）
  2. 写入 `auditMode=auto` + `auditedAt=now` + `auditedBy=system-auto`
  3. **同步**调用项目协同空间创建接口（详见"项目空间创建规则"）
  4. **同步**派发撮合事件到撮合引擎
  5. 不向运营审核队列推送消息
- 影响范围：仅套餐模式；自定义模式不适用
- 审计追溯：操作日志中记录"系统自动审核通过 · 套餐模式 · SKU={skuId} · terms_version={terms_version}"

**项目协同空间创建规则**：
- 触发条件：状态首次进入"已审核通过"分支（套餐 `draft → auto-approved` / 自定义 `auditing → matching`）
- 调用方式：本模块作为调用方，**同步**调用协同模块 `POST /api/project-spaces`
- 入参：`demandId` + `workspace` + `createdFrom`（`AUTO_APPROVED` / `MANUAL_APPROVED`）+ 必要的需求快照引用
- 出参：`projectSpaceId`
- 系统动作：
  - 成功 → 写入需求单 `projectSpaceId` + `projectSpaceCreatedAt`
  - 失败 → **不阻塞主流程**，需求单状态保持 `auto-approved` / `matching`，由后端补偿任务异步重试（详见 §17.9）
- 幂等保障：协同模块基于 `demandId` 幂等，重复调用返回同一 `projectSpaceId`
- 不允许场景：
  - 同一需求单创建多个项目空间（违反 1:1 约束）
  - `rejected` / `cancelled` 状态的需求单创建项目空间
  - 需求单删除前未归档项目空间（一期不涉及，二期"关闭需求"流程联动）

### 12.2 配置规则

**类目配置**：
- 是否可配置：是（后端管理，前端消费）
- 配置粒度：一级类目级
- 生效时机：前端加载配置接口时生效
- 对历史数据是否生效：否，已提交需求不受配置变更影响
- 一期类目配置硬编码在前端，二期迁移到后端配置接口

**SPU/SKU 目录配置**：
- 是否可配置：是（运营管理后台，本期不涉及 UI）
- 配置粒度：SKU 级（含价格、数量、交付规格、条款版本）
- 生效时机：前端加载目录接口时生效
- SPU 可标记 `upcoming=true` 表示即将上线（前端置灰不可选）

### 12.3 配置生效验证机制

- 一期配置硬编码在前端，不涉及草稿/发布版本区分
- 二期迁移到后端后**必须**：
  - **区分草稿态与线上态**：运营编辑流程为「编辑草稿 → 预览 → 发布上线」。已生效的配置不允许直接覆盖
  - 配置保存为草稿不影响线上，仅在"发布上线"动作后生效
  - 缓存刷新时效：配置发布后前端通过 API 返回的版本号检测变更，主动刷新本地缓存
  - 配置变更不影响已提交的需求（已落库数据为彼时配置的快照）
  - SPU/SKU 目录与类目配置共享同一套版本管理范式

## 14. 技术约束与非功能需求

- 预估数据量级：单用户需求量 <100（一期），全平台预估 <10,000（一期）
- 峰值并发 / QPS：需求提交 <10 QPS `[AI 推测 - 待业务确认]`
- 页面响应时效要求：发布页加载 <2s，列表页加载 <1s
- 列表查询时效要求：<500ms
- **数据保留周期**：
  - 需求数据 **永久保留（逻辑删除），不设 TTL**
  - 理由：需求是交易链路源头，关联订单/报价/结算等下游数据，物理删除会导致数据断链
  - 归档策略：`completed` 状态超过 2 年的需求由后端运维迁移至冷存归档，仍可通过 ID 检索
- 日志保留周期：操作日志 ≥1 年 `[AI 推测 - 待业务确认]`
- SKU 快照不可修改、不可删除，作为订单条款追溯依据

## 15. 上下游依赖与一致性

| 方向 | 系统/模块 | 依赖方式 | 一致性要求 |
|---|---|---|---|
| 上游 | 入驻认证模块 | 读取 `profileComplete` 状态 | 强一致 |
| 上游 | 团队配置 | 读取当前团队信息（canPublish, payerLabel 等） | 强一致 |
| 下游 | 运营审核模块 | 自定义模式提交后通知审核队列；**套餐模式不通知** | 最终一致 |
| 下游 | **项目协同空间模块** | **审核通过后同步调用 `POST /api/project-spaces`，写回 `projectSpaceId`** | **最终一致**（同步失败由补偿任务保障） |
| 下游 | 撮合引擎 | 审核通过后触发（套餐自动通过即触发，自定义人工通过时触发） | 最终一致 |
| 下游 | 订单模块 | 选定承制方后创建订单 | 强一致 |
| 下游 | 钱包模块 | 订单创建时执行锁资 | 强一致 |

- 同步失败是否阻塞主流程：
  - 入驻状态检查失败 → **阻塞提交**（强一致依赖）
  - 空间余额查询失败 → **不阻塞**，摘要面板余额显示"—"
  - 审核通知失败 → **不阻塞**，详见 §8.4：MQ 异步推送 + 重试 3 次（指数退避 5s / 30s / 120s）+ 死信队列 + 告警
  - **项目协同空间创建失败** → **不阻塞**，需求状态保持 `auto-approved` / `matching`，详见 §17.9：补偿任务异步重试 + DLQ + 告警
- 锁资失败时：钱包页可通过深链 `?from=wallet&demandId=xxx` 跳回发布页调整金额

| `/api/categories` | GET | 获取类目配置 | — | CategoryConfig 树 + version | — |

> **项目空间创建接口调用契约**（本模块作为调用方需保障）：
> - 调用时机：套餐模式 → `POST /api/demands` 写入成功后立即调用；自定义模式 → 审核员审核通过事件触发后调用

> - 成功回写：`projectSpaceId` + `projectSpaceCreatedAt` 写入需求单
> - 列表页前端基于 `projectSpaceId` 是否存在判断"进入项目空间"按钮可点状态

## 17. 异常场景与边界处理

### 17.2 发布页 · 提交校验失败

| 场景 | 触发条件 | 处理方式 | 用户数据是否保留 |
|---|---|---|---|
| 必填项缺失（自定义模式） | 类目/标签/等级/方式/专属字段/标题/描述/预算/交期 任意缺失 | Toast 列出**所有**缺失项（不是只报第一项），如"还缺：类目、至少 1 个标签、服务等级 + 制作方式" | 全部保留，不清空 |

| 标题超长 | 超过 40 字 | 前端 maxlength=40 限制输入，无需 Toast | — |
| 描述超长 | 超过 500 字 | 前端 maxlength=500 限制输入 | — |
| 参考链接格式错误 | 非合法 URL | 一期**不做格式校验**（参考链接为给承制方看的辅助信息，可能是网盘/内网/纯文字说明，强校验会误伤合法用例）。二期可加"温和提示"（检测到非 URL 形态时提示一句但不阻断） |  — |

### 17.3 发布页 · 权限与准入拦截

| 场景 | 触发条件 | 处理方式 |

| 切换到管理员视角（页面内） | 在发布页填写过程中通过顶部栏切换到 cf-admin | 立即刷新阻断状态：显示横幅 + 禁用按钮；已填数据保留在 DOM 中但不可提交。Toast "提示：当前团队不参与发单，请切换到业务团队" |
| 切换回业务空间 | 从 cf-admin 切回 cf-drama 等 | 阻断横幅消失、按钮恢复可用；但需检测未配预算空间的回退提示 |
| 入驻资料未完善 | profileComplete = false | 点击"提交需求"时弹出全屏遮罩弹窗，含三个按钮：「稍后再说」关闭弹窗 / 「立即完善」跳转入驻页 / 「Demo 跳过」标记完善并继续。不执行提交，已填数据保留 |
| 入驻资料未完善 + 暂存草稿 | profileComplete = false，点击暂存 | **不拦截**，允许暂存草稿（草稿不涉及交易） |
| 用户角色无 demand.publish 权限 | 企业成员但角色为纯财务等 | 一期前端**不做角色级权限校验**（所有需求方身份用户均可进入发布页）。二期补充：进入发布页时校验角色，无权限时显示阻断态"您的角色无发布权限，请联系企业管理员"，按钮禁用 |
| 登录态过期 / session 失效 | 长时间填写后提交 | 前端在 401 响应处理中、跳转登录页**前先调用 `setDraft(form)` 自动暂存草稿**，跳转登录页。登录回来后进入发布页自动恢复草稿，体验闭环 |

### 17.4 发布页 · 模式切换与类目切换

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| 套餐 → 自定义切换 | 已选 SPU+SKU 后切换到自定义 | 清空 spuId / skuId / skuSnapshot + 隐藏套餐区域；已填的标题/描述/交期/参考资料**保留**（两种模式共享 Step 5） |
| 自定义 → 套餐切换 | 已选类目+标签+等级后切换到套餐 | 清空 l1 / l2 / tags / level / methods / dyn + 隐藏自定义步骤；已填的标题/描述/参考资料**保留** |
| 切换一级类目 | 已选"视频制作"的所有标签，切换到"文案服务" | **全部清空**：二级类目、标签、服务等级、制作方式、专属字段值。闪光动画提示 Step 2/3/4 已切换。Toast "已选择「文案服务」，下面 4 个区块已切换为该类目专属表单" |
| 反复切换一级类目 | 快速在不同类目间切换 | 每次切换都完全重置下级数据，无脏数据残留。前端需保证 DOM 完全重建（非复用旧节点） |

### 17.5 发布页 · 草稿暂存与恢复

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| 正常暂存 | 点击"暂存草稿" | 序列化当前完整 form 状态到 localStorage（含 publishMode / spuId / skuId / l1 / l2 / tags / level / methods / dyn / title / desc / budgetMin / budgetMax / deadline / refs）。Toast "草稿已暂存到本地" |
| localStorage 存储空间不足 | 浏览器 localStorage 容量已满 | try-catch 捕获 `QuotaExceededError` → Toast "暂存失败：浏览器存储空间不足，请清理后重试" → **不阻塞提交和其他操作**（草稿 <10KB 触发概率极低，不应因此影响主流程） |
| 草稿数据损坏 | localStorage 中 JSON 解析失败 | try-catch 捕获 JSON.parse 异常 → 忽略草稿，正常进入空表单，不 Toast 错误（静默降级） |
| 旧版草稿无 publishMode 字段 | 从旧版本升级后恢复草稿 | 若草稿中存在 l1 数据 → 推断 publishMode = "custom"；否则 → 默认 "package" |
| 草稿中的 spuId/skuId 有效但 skuSnapshot 缺失 | 旧版草稿未保存快照 | 从当前 SPU/SKU 目录重新获取对应 SKU 并生成快照；若 SKU 已下架 → 清空套餐选择，Toast "之前选择的套餐档位已下架，请重新选择" |
| 草稿恢复后 SKU 已调价 | 草稿中快照价格 = ¥30,000，但当前目录该 SKU 已调价为 ¥35,000 | 恢复时从目录重新获取 → **用最新价格覆盖快照** + Toast "套餐「进阶档」价格已更新为 ¥35,000"。理由：旧价格非平台当前有效报价，提交时后端校验必然不通过；用户必须感知价格变化。若不接受新价可切换到自定义模式自拟预算 |
| 草稿中引用的类目配置已变更 | 草稿中的标签/等级/专属字段与当前配置不匹配 | 一期类目配置硬编码不变化，此场景暂不触发。二期迁移后端后需设计配置版本比对机制 |
| 提交成功后草稿未清除 | 提交后 clearDraft 失败 | try-catch 处理，即使清除失败也不阻塞提交成功流程。下次进入发布页时，恢复草稿后用户重新填写即可（草稿幂等） |
| 多浏览器标签页编辑 | 用户在两个标签页同时编辑发布页 | 两个标签页共享同一份 localStorage 草稿。后暂存的覆盖先暂存的，恢复时取最后一次暂存。不做冲突提示（一期可接受） |
| 浏览器意外关闭 | 填写过程中崩溃/关机 | 已暂存的草稿在下次进入时恢复；未暂存的数据丢失。**二期补自动暂存**：表单变更后 debounce 5 秒触发一次 `setDraft`（不采用 60 秒定时器，因用户停手 60 秒说明不在填写，无需频繁存储） |

### 17.6 发布页 · 团队与资金感知

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| 团队未配预算 | 当前团队 = cf-ip（budgetMissing=true） | 允许发布。摘要面板显示"⚠️ 团队未配预算 → 回退企业主账户"。切换到该团队时 Toast "提示：该团队未配置预算，下单将回退至企业主账户" |
| 套餐价格超出空间余额 | 选中 SKU 价格 = ¥40,000，空间余额 = ¥3,200 | **一期不在发布侧做余额预检**（实际锁资在订单创建时由钱包模块执行）。理由：余额动态变化（充值/解锁随时发生），发布时余额 ≠ 下单时余额，预检不保证准确，且增加额外接口调用。**二期**在摘要面板增加"当前余额：¥XXX"灰色小字辅助判断（不阻断发布），锁资失败已有完整深链兜底 |
| 自定义预算超出空间余额 | 填写预算最高值 = ¥500,000，空间余额 = ¥185,000 | 同上，一期不拦截 |
| 钱包页深链跳入 | URL 含 `?from=wallet` | Toast "📍 从钱包页跳入需求编辑 · 锁资失败后可调整金额或追加预算"。若含 `demandId` 参数，提示具体需求号 |
| 钱包页深链 + 需求 ID 不存在 | `?demandId=DM-9999` 但该需求不存在 | Toast 仍显示（仅为来源提示），不影响正常发布流程 |
| 发布过程中空间被冻结/删除 | 管理员在后台冻结了当前团队 | **一期仅在提交时由后端校验空间有效性**（不做实时感知，因填表过程中空间被冻结是极低概率事件，不值得做轮询/WebSocket）。后端返回 400 "空间不可用"前端 Toast "当前团队不可用，请切换空间后重试"，已填数据保留 |

### 17.7 发布页 · 网络与接口异常

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| 页面加载失败 | 配置接口 / 目录接口超时 | 一期配置硬编码在前端，不存在此问题。二期从后端加载后需处理：显示全页错误态 + "重新加载"按钮 |
| 提交接口超时 | POST /api/demands 超时 | 前端恢复提交按钮可点击状态，Toast "提交超时，请重试"。已填数据不清空。后端需保证幂等（防止超时后用户重试导致重复创建） |
| 提交接口返回 500 | 服务端内部错误 | Toast "提交失败，请稍后重试"。已填数据不清空。按钮恢复可用 |
| 提交接口返回 400 | 业务校验失败（后端二次校验） | Toast 展示后端返回的错误信息（如"该类目已下架""预算超出限额"等） |
| 提交接口返回 401 | 登录态过期 | 前端**先调用 `setDraft(form)` 暂存当前表单状态**，然后跳转登录页。登录回来后自动恢复草稿 |
| 提交接口返回 403 | 权限不足 | Toast "您无权发布需求，请联系管理员" |
| 提交接口返回 409 | 重复提交（幂等拦截） | Toast "该需求已提交，请勿重复操作" + 跳转列表页 |
| 列表接口超时 | GET /api/demands 超时 | 列表区显示加载失败态 + "重新加载"按钮。状态聚合卡片显示"—" |
| 列表接口返回空数组 | 无需求数据 | 显示空状态插画："📭 没有符合条件的需求" + "→ 立即发布一个新需求"链接 |

### 17.8 列表页 · 筛选与展示异常

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| 筛选后无结果 | 切换状态/空间/类目/搜索后列表为空 | 显示空状态 + 链接。状态聚合卡片仍按筛选范围统计（不因搜索关键词变为 0） |
| highlight 参数指向不存在的需求 | `?highlight=DM-XXXX` 但该需求不在列表中 | 静默忽略（不 Toast 错误），正常渲染列表 |
| highlight 参数指向被筛选掉的需求 | 该需求存在但被当前筛选条件过滤 | **自动 reset 筛选条件为"所有团队 + 全部状态"** 以确保高亮可见。理由：highlight 触发场景是"刚提交完跳转回来"，看不到自己的新需求体验极差，reset 实现成本极低 |
| 需求状态在查看期间被变更 | 审核员在后台通过了某需求，用户正在看列表 | **一期列表不自动刷新**（需用户手动刷新或切换筛选触发重新渲染）。理由：需求状态非秒级敏感，WebSocket / 轮询的工期不值得在一期投入。二期视用户反馈决定是否补 |
| 需求所属团队已被删除 | 空间被管理员删除但需求仍在 | 空间标签显示原空间名称（冗余字段 workspaceName），不显示动态样式。操作按钮正常渲染 |
| 操作按钮跳转目标页不存在 | 如 demand-quote-comparison.html 尚未开发 | **改为 Toast "功能开发中，敬请期待"**（不允许跳转 404，跳转 404 体验极差） |
| 大量需求导致渲染卡顿 | 单用户需求 >200 条 | 一期全量渲染（预估不会触发）。二期阈值触发（单用户均值 >100 或 P95 >200）后启用服务端分页 + 虚拟滚动 |

### 17.9 跨模块同步异常

| 场景 | 上游/下游 | 处理方式 |
|---|---|---|
| 提交后审核通知失败（自定义模式） | 下游：运营审核模块 | **不阻塞提交**。需求状态正常置为 auditing。后端 MQ 异步推送审核队列，重试 3 次（指数退避 5s / 30s / 120s），超限写入死信队列（DLQ）+ 告警运营群，由人工补处理 |
| 套餐模式提交后项目空间创建失败 | 下游：项目协同空间模块 | **不阻塞提交**。需求状态正常置为 `auto-approved`，`projectSpaceId` 暂为 null。后端写入"待补偿"表，异步重试 3 次（指数退避 5s / 30s / 120s），超限入 DLQ + 告警；列表页"进入项目空间"按钮在 `projectSpaceId` 为空时灰显 + tooltip "项目空间正在创建中..."。**SLA 要求：99% 的需求空间在 5 分钟内创建成功，所有需求必须在 24 小时内补齐**（详见 §6.1） |
| 自定义模式审核通过后项目空间创建失败 | 下游：项目协同空间模块 | 同上。需求状态正常置为 `matching`，`projectSpaceId` 暂为 null，由补偿任务异步重试 |
| 套餐模式提交后撮合事件派发失败 | 下游：撮合引擎 | **不阻塞提交**。需求状态仍为 `auto-approved`。同审核通知失败兜底：MQ 重试 3 次 + DLQ + 告警 |
| 审核通过后撮合引擎不可用 | 下游：撮合引擎 | 需求状态仍为 matching / auto-approved（审核已通过）。撮合引擎恢复后自动处理积压。需求方看到"撮合中"状态属正常等待 |
| 选定承制方后锁资失败 | 下游：钱包模块 | 不回退需求状态（需求仍为 quoting）。钱包页展示锁资失败原因 → 深链跳回发布页调整金额 / 跳转钱包充值页追加预算 |
| 入驻状态查询失败 | 上游：入驻认证模块 | 阻塞提交。Toast "系统异常，无法校验入驻状态，请稍后重试" |
| 空间余额查询失败 | 上游：钱包模块 | 一期摘要面板余额显示依赖页面加载时获取，查询失败则显示"—"，不阻塞发布 |
| 团队配置查询失败 | 上游：空间模块 | 回退到默认空间（cf-drama）。若默认空间也不可用 → 全页错误态 |
| 项目空间创建接口超时（>3s） | 下游：项目协同空间模块 | 主流程**不等待**，同步调用进入超时熔断后立即返回成功响应给客户端（需求已落库），后台并行走异步补偿链路 |
| 项目空间已创建但 `projectSpaceId` 回写失败 | 下游：项目协同空间模块 | 协同模块基于 `demandId` 幂等，补偿任务再次调用会返回相同 `projectSpaceId`，回写成功 |

### 17.10 数据一致性异常

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| SKU 在用户选择后、提交前被下架 | 运营在后台下架了该 SKU | 一期配置硬编码在前端，此场景不触发。二期从后端加载后：提交时后端校验 SKU 有效性 → 返回 400 "该套餐档位已下架，请重新选择" → 前端清空 SKU 选择 |
| SKU 在用户选择后、提交前被调价 | 运营调整了 SKU 价格 | 快照是选定时生成的深拷贝。提交时后端比对快照 `terms_version` 与当前 SKU `terms_version` → **不一致返回 409 + "套餐条款已更新，请刷新页面后重新选择"**。前端**清空 SKU 选择**，但**保留标题/描述/参考资料等其他数据**，减少用户重填成本。理由：terms_version 变化意味着服务范围/交付规格/法律条款可能都变了，不能基于旧条款下单 |
| 类目配置变更后用户仍在旧配置页面 | 后端更新了类目配置 | 一期硬编码不触发。二期建议 API 返回配置版本号，前端提交时附带版本号，后端校验版本一致性 |
| 需求 ID 重复 | 并发生成相同 ID | **由后端统一生成**，格式 `DM-{YYYYMMDD}{6位自增序号}`（示例：`DM-20260507000001`）。并发量 <10 QPS，使用 DB sequence 或 Redis INCR 即可保证全局唯一。前端 ID 生成仅用于 demo 原型，生产环境必须由后端生成 |
| 需求数据与空间数据不一致 | 需求记录的 workspace 在空间表中不存在 | 列表页使用冗余字段 workspaceName 展示，不依赖空间表实时查询。若冗余字段也缺失 → 显示 workspace ID 原值 |
| 套餐模式需求被错误置为 `auditing` | 后端逻辑 bug 或灰度过程中 | 数据修复脚本：扫描 `requirementType=PLATFORM_SKU` 且 `status=auditing` 的需求，修正为 `auto-approved` 并补齐 `auditMode/auditedAt/auditedBy` 字段；触发项目空间创建补偿 |
| 同一需求单存在多个项目空间 ID | 协同模块幂等失效（极小概率） | 后端定时巡检：对 `projectSpaceId` 字段去重，若发现协同模块存在多条对应同一 `demandId` 的项目空间记录，保留最早创建的，其余标记冗余并通知协同模块清理 |
| 需求已审核通过但 `projectSpaceId` 长期为空 | 项目空间创建链路全部失败 | 后端定时巡检：对 `status ∈ {auto-approved, matching, quoting, in-progress}` 且 `projectSpaceId IS NULL` 且 `auditedAt < now() - 5min` 的需求触发主动重建；超过 24 小时仍未恢复则告警人工介入 |

### 17.11 并发冲突策略

- 发布需求为**单用户单端操作**，一期不涉及多人并发编辑同一需求
- 提交时**必须防重复提交**：
  - 前端：按钮点击后立即 `disabled` + 文案改为"提交中..."
  - 后端幂等：基于 `userId + SHA256(title + desc + workspace + requirementType)` 组合去重
  - 幂等窗口期：**30 秒**（足以覆盖网络抖动 / 双击 / 快速重试，不会误伤新需求）
  - 窗口内重复提交：返回 409 + **第一次创建的 `demandId`**，前端正常跳转列表页，对用户视角等价于"提交成功"
- 草稿暂存无并发冲突风险（localStorage 单线程写入，后暂存覆盖先暂存）
- 列表页筛选为纯读操作，无并发冲突

### 17.12 逆向动作失败与补偿

- 一期需求模块自身**不涉及逆向资金动作**（锁资/退款由钱包模块处理）
- 驳回后修改重提为**正向流程**（PATCH 更新需求数据 + 状态回 auditing），不涉及逆向回退
- **`auditing → cancelled` 撤回流程**（一期支持，详见 §8.2 / §8.3）：
  - 撤回后通过 MQ 通知审核队列移除该需求
  - 需求状态变为 `cancelled`，**不可恢复**（需重新发布）
  - 不涉及资金冲抵（auditing 阶段尚未锁资）
  - 列表页该需求显示"已取消"灰色徽章，仅"查看归档"操作
- **`quoting → cancelled` 关闭流程**（二期能力）：
  - 触发时通知所有已报价承制方"需求已关闭"
  - 锁资前关闭：不涉及资金冲抵（锁资发生在选定承制方时）
  - 锁资后关闭：联动钱包模块执行解锁（属于跨模块逆向流程，二期联动设计）

## 20. 验收标准

### 20.1 主流程验收

**套餐模式**：
- 给定：需求方以"短剧制作中心"空间登录
- 当：选择"选平台套餐" → 选择"营销短视频制作包" → 选择"进阶档 · 164 条" → 填写标题和描述 → 点击提交
- 则：
  - 需求 ID 生成且格式正确
  - `requirementType = PLATFORM_SKU`
  - `skuSnapshot` 包含完整 SKU 数据（含 spu_id, spu_name, terms_version）
  - `budget = 30000`（SKU 锁价）
  - `workspace = cf-drama`
  - **`status = auto-approved`**（不经过 auditing）
  - **`auditMode = auto`、`auditedBy = system-auto`、`auditedAt = createTime`（瞬时）**
  - **`projectSpaceId` 非空**（同步创建成功）或暂为 null（已派发补偿任务，5 分钟内补齐）
  - **Toast 显示"需求已自动通过审核 · 项目空间已创建"**
  - 页面跳转至列表页，新需求出现在列表顶部
  - 列表行状态徽章显示"撮合中"+ 自动通过 ✓ 标识，"进入项目空间"按钮可点（项目空间 ID 已写入）

**自定义模式**：
- 给定：需求方以"市场宣传组"空间登录
- 当：切换到"自定义发布" → 选择"文案服务 → 营销文案" → 选择标签 → 选择等级和方式 → 填写专属字段 → 填写标题/描述/预算/交期 → 提交
- 则：
  - `requirementType = CUSTOM`
  - `l1 = copy`, `l2 = marketing`
  - `tags`, `level`, `methods`, `dyn` 均有值
  - `budgetMin ≤ budgetMax`
  - `workspace = cf-marketing`
  - **`status = auditing`、`auditMode = manual`、`auditedAt = null`、`projectSpaceId = null`**

**自定义模式审核通过后**：
- 给定：自定义模式需求处于 `auditing` 状态
- 当：审核员在后台审核通过
- 则：
  - `status = matching`
  - `auditMode = manual`、`auditedBy = 审核员 ID`、`auditedAt = 审核完成时间`
  - 同步触发项目空间创建，成功后 `projectSpaceId` 非空、`projectSpaceCreatedAt` 写入
  - 列表页"进入项目空间"按钮可点

- 给定：用户在 30 秒内连续两次提交完全相同的需求（同 userId、同 title、同 desc、同 workspace、同 requirementType）
- 当：第二次提交到达后端
- 则：后端返回 409 + 第一次创建的 `demandId`，前端表现为正常跳转列表页（用户视角等价于"提交成功"），数据库中仅有一条记录

- 给定：用户在不同空间发布同名需求
- 当：先后提交两次（workspace 不同）
- 则：两次提交均成功，生成两条独立需求

### 20.11 套餐自动审核与项目空间创建验收

- 给定：套餐模式需求提交成功
- 当：查看需求审计日志
- 则：日志中存在"系统自动审核通过"记录，含 SKU 快照 ID、terms_version、auditedAt 字段；不存在该需求的人工审核记录

- 给定：套餐模式需求提交时项目协同空间创建接口不可用
- 当：提交完成
- 则：
  - 需求状态正常置为 `auto-approved`，主流程不阻塞
  - `projectSpaceId` 暂为 null
  - 列表页"进入项目空间"按钮显示为灰显态 + tooltip "项目空间正在创建中..."
  - 后端补偿任务在 5 分钟内重试创建成功，列表刷新后按钮恢复可点
  - 操作日志中存在创建失败 + 重试成功的完整记录

- 给定：自定义模式需求审核通过后项目空间创建失败
- 当：触发补偿任务
- 则：补偿任务调用协同模块时，协同模块基于 `demandId` 幂等，返回（重新生成或返回已存在的）`projectSpaceId`，回写到需求单成功

- 给定：套餐模式需求 `status = auto-approved`
- 当：查看列表页该需求行的可用操作
- 则：可用操作与 `matching` 状态完全一致（"进入项目空间" + "详情"），不显示"撤回"按钮

- 给定：套餐模式需求处于 `auto-approved` 状态
- 当：用户尝试调用撤回接口
- 则：后端返回 409 "当前状态不可撤回"（一期 auto-approved 不支持撤回）

- 给定：状态聚合卡片
- 当：统计"撮合中"卡片数量
- 则：数量 = `matching` 状态需求数 + `auto-approved` 状态需求数（合并统计）
