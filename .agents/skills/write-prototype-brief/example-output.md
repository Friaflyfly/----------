# 接单平台｜项目大厅页 Prototype Brief（示例）

## 1. 页面名称
- 页面名称：项目大厅页
- 页面类型：列表页 / 运营型任务市场页
- 所属模块：接单平台
- 所属业务链路：项目发布 → 学员找单 → 接单 / 报名 → 进入协同空间

---

## 2. 页面目标
- 帮助学员快速浏览平台当前可参与的项目，并根据自身能力与偏好完成筛选、判断和接单 / 报名。
- 页面成功的标志是：用户能在 3 步内完成“发现项目 → 判断是否适合 → 发起接单 / 报名”。
- 本页不解决的问题：
  - 项目详细需求的完整承接在“项目详情页”
  - 接单后的协同、上传、审核、验收在“协同空间页”
  - 榜单规则配置和项目上下架在后台页面处理

---

## 3. 使用角色
| 角色 | 可见区块 | 可见字段 | 可操作按钮 | 特殊权限 / 限制 |
|---|---|---|---|---|
| 学员 / 接单方 | 全部前台区块 | 项目基础信息、标签、难度、交付周期、接单模式、名额、榜单 | 查看详情、接单 / 报名 | 仅可对符合资格的项目发起接单 / 报名 |
| 游客 / 未登录用户 | 列表区、部分榜单信息 | 可见公开项目信息 | 查看详情、去登录 | 不可直接接单 / 报名 |
| 已冻结 / 暂停接单学员 | 列表区、详情页 | 与普通学员一致 | 查看详情 | 接单按钮置灰并提示原因 |
| 管理员 / 运营 | 不在本页主角色范围 | 前台不暴露管理能力 | 无 | 管理操作在后台执行 |

---

## 4. 页面入口 / 出口
- 页面入口：
  - 首页导航「项目大厅」
  - 顶部导航「接单」
  - 榜单页点击“去找项目”
- 页面出口：
  - 点击项目卡片进入项目详情页
  - 点击“接单 / 报名”后：
    - 抢单模式：成功进入协同空间或订单详情
    - 报名模式：停留当前页并更新项目状态 / 我的报名状态
- 面包屑路径：
  - 首页 > 接单平台 > 项目大厅
- 多入口返回逻辑：
  - 统一返回项目大厅，不依赖来源页
- 页面层级：
  - 一级业务页，不应作为抽屉打开

---

## 5. 页面信息架构
页面整体采用 **“顶部导航区 → 筛选区 → 榜单切换 / 推荐区 → 项目列表区 → 分页区”** 的结构。

### 5.1 页面区块顺序
1. 页面标题区
2. 搜索筛选区
3. 榜单 / 维度切换区
4. 项目列表区
5. 分页区

### 5.2 主区块
- 搜索筛选区
- 项目列表区

### 5.3 辅助区块
- 榜单维度切换
- 空态提示
- 无权限提示

---

## 6. 核心模块说明

### 6.1 搜索筛选区
**模块目标：**  
帮助用户快速定位适合自己的项目，降低找单成本。

**模块内容：**
- 搜索框：按项目名称关键词搜索
- 分类筛选：短剧 / AI 漫剧 / 短视频 / 综艺
- 标签筛选：如霸总、穿越、男频、女频等
- 难度筛选：精品 / 工厂
- 接单模式筛选：抢单 / 报名
- 状态筛选：可接单 / 已满 / 已截止
- 截止时间筛选：近 3 天 / 近 7 天 / 自定义

**模块操作：**
- 输入关键词实时回车查询
- 切换筛选项自动刷新列表
- 点击“重置”恢复默认筛选条件

**默认态逻辑：**
- 默认展示：全部分类 + 全部标签 + 全部难度 + 可接单 + 近 30 天内项目
- 点击“重置”后恢复上述默认态，而不是简单清空所有输入
- 重置后自动重新查询，并回到第一页

---

### 6.2 榜单 / 维度切换区
**模块目标：**  
增强找单信心和平台活跃感，引导用户发现热门项目与优秀学员。

**模块内容：**
- 维度切换：总接单量 / 总收益 / 近期活跃度
- 榜单时间范围：总榜 / 月榜
- “查看完整榜单”入口

**模块操作：**
- 切换维度后刷新榜单卡片
- 点击榜单成员进入个人主页（如后续支持）

**状态与视觉：**
- 当前选中维度需高亮
- 榜单前三名可用强化样式

---

### 6.3 项目列表区
**模块目标：**  
承接用户完成浏览、判断与发起操作。

**模块内容：**
每个项目卡片 / 行项至少包含：
- 项目名称
- 固定标签
- 内容类别
- 难度
- 接单模式
- 交付周期
- 截止时间
- 集数
- 名额 / 已接人数
- 当前状态
- 操作按钮

**模块操作：**
- 点击卡片主体进入详情页
- 点击按钮执行接单 / 报名
- hover 展示更多摘要信息（如项目说明前 80 字）

**排序逻辑：**
- 默认按推荐排序
- 可切换按单价、截止时间、最新发布排序
- 截止时间支持升序，其他默认降序

**行级操作规则：**
- 操作按钮不超过 2 个：查看详情、接单 / 报名
- 不需要“更多”收纳

---

### 6.4 分页区
**模块目标：**  
承载长列表翻页操作。

**分页规则：**
- 默认每页 20 条
- 支持切换 20 / 50 / 100
- 切换筛选条件后回到第一页
- 当前页码、排序、筛选条件需写入 URL 参数

## 7. 字段与数据展示规则

### 7.1 列表字段规则
| 字段 | 含义 | 展示位置 | 类型 | 必填 | 可编辑 | 备注 |

| difficulty | 难度 | 基础信息区 | 枚举 | 是 | 否 | 精品 / 工厂 |
| take_mode | 接单模式 | 基础信息区 | 枚举 | 是 | 否 | 抢单 / 报名 |
| due_time | 截止时间 | 基础信息区 | 日期时间 | 是 | 否 | 超时后状态自动变化 |

| action_button | 操作按钮 | 右侧操作区 | 按钮 | 是 | 否 | 接单 / 报名 / 查看详情 |

### 7.2 表格 / 列表视觉规范
- 文本左对齐
- 数字 / 集数 / 名额右对齐
- 状态标签居中
- 操作按钮居中
- 长文本默认超出隐藏（Ellipsis）
- hover 通过 Tooltip 展示全量
- 标签区超出时折叠为 `+N`

### 7.3 搜索字段规则
| 字段 | 类型 | placeholder | 默认值 | 备注 |
|---|---|---|---|---|
| keyword | 文本 | 搜索项目名称 | 空 | 回车触发查询 |
| category | 下拉单选 | 请选择分类 | 全部 | 切换自动查询 |
| tag | 多选标签 | 请选择标签 | 空 | 支持多选 |
| difficulty | 下拉单选 | 请选择难度 | 全部 | 切换自动查询 |
| take_mode | 下拉单选 | 请选择接单模式 | 全部 | 切换自动查询 |
| deadline_range | 预设 / 日期范围 | 请选择截止时间 | 近 30 天 | 默认态字段 |

## 9. 关键交互与状态

### 9.1 主交互
- 点击项目卡片主体：进入项目详情页
- 点击“立即接单”：
  - 校验登录状态
  - 校验学员状态
  - 校验并发上限
  - 校验项目是否可接单
  - 校验是否已接过同一项目
- 点击“立即报名”：
  - 校验登录状态
  - 校验学员状态
  - 校验是否已报名
  - 成功后按钮态变为“已报名（审核中）”

### 9.2 按钮置灰条件
- 学员状态非可接单
- 项目状态非 available
- 名额已满
- 已接单 / 已报名
- 项目已截止

置灰时 hover 需要提示原因，例如：
- “当前账号不可接单，请联系运营”
- “该项目报名已结束”
- “你已参与过该项目”

### 9.3 成功反馈
- 接单成功：Toast“接单成功，正在进入项目空间”，随后跳转
- 报名成功：Toast“报名成功，请等待审核”
- 切换筛选成功：列表刷新并回到第一页

### 9.4 失败反馈
- 网络失败：顶部错误提示 + 可重试
- 校验失败：就地提示原因
- 并发超限：Toast“当前接单数已达上限”
- 重复报名：Toast“你已报名该项目，请勿重复操作”

### 9.5 高危 / 特殊操作
本页无删除类高危操作。  
若未来增加“撤回报名”，需补充二次确认弹窗：
- 标题：确认撤回报名？
- 正文：撤回后将退出该项目报名队列，需重新报名才可参与。
- 确认按钮：确认撤回
- 取消按钮：取消

### 9.6 副作用说明
- 切换筛选条件后，列表区、分页区、榜单推荐位需同步刷新
- 报名成功后，当前项目卡片按钮态需立即更新
- 接单成功后，项目大厅返回时应同步更新该项目的剩余名额与我的参与状态
- 若大厅页使用缓存，接单 / 报名成功后应使当前列表缓存失效并重新拉取

## 10. 异常态 / 权限态 / 空态

### 10.1 空态
- 无数据时展示空态插图 + 文案：
  - “暂无符合条件的项目”
  - 辅助文案：“试试调整筛选条件或稍后再来”
- 提供“重置筛选”按钮

### 10.2 加载态
- 首屏使用骨架屏，而不是全局 loading
- 骨架屏覆盖：
  - 标题区
  - 筛选区
  - 3~5 个项目卡片骨架
  - 分页区

### 10.3 加载失败
- 展示错误态区块：
  - 文案：“项目加载失败，请稍后重试”
  - 按钮：“重新加载”

### 10.4 权限态
- 未登录：可浏览列表，点击接单 / 报名时弹登录引导
- 已冻结 / 暂停：按钮置灰，并展示说明
- 无权限：若用户无接单资格，展示“当前账号暂无接单权限”

## 13. 给 AI coding 工具的实现提示
- 页面优先采用 **PC 端 1440px 宽度保真**
- 布局建议：顶部筛选区 + 卡片式项目列表
- 先生成静态 HTML + 基础交互，不接真实接口
- 使用 mock 数据驱动列表渲染
- 需要实现的基础交互：
  - 筛选项切换
  - 重置恢复默认态
  - 排序切换
  - 按钮置灰与 hover 提示
- 建议状态变量命名：
  - `projectStatus`
  - `takeMode`
  - `applyStatus`
  - `isDisabled`
  - `filters`
- 建议 HTML 使用语义化标签：
  - `<main>`
  - `<section>`
  - `<nav>`
  - `<article>`
- 基础 A11y 要求：
  - 按钮提供 `aria-label`
  - 图标或图片提供 `alt`
  - 搜索输入框提供明确 label
  - 状态不能只靠颜色表达
- 建议使用多样化 mock 数据验证布局稳定性，包括：
  - 极限长项目名称
  - 空标签
  - 不同长度的项目描述
  - 所有状态样本（available / full / expired / pending）
  - 名额为 0、1、10、100 的样本
