# 接单平台 1.0｜一期 MVP PRD（示例输出）

## 1. 需求背景

当前项目制内容交付主要依赖表格和群聊推进，存在以下核心问题：
1. 项目、订单、进度分散，缺少统一状态流转与责任留痕。  
2. 素材与成品缺少版本管理，多轮修改不可追溯。  
3. 运营缺少项目台账、人员管理、统一审核与对账能力，难以规模化运营。  
4. 榜单、收益、对账缺乏统一账本依据，口径不一致。  

因此，需要建设一个最小可运行的接单平台，跑通“发布 → 接单 → 交付 → 审核 → 验收 → 入账 → 对账 / 结算”闭环，并形成后台运营能力。

## 2. 需求目标

### 2.1 本期目标
1. 学员端支持分类找单、一键接单、在线协同交付。  
2. 订单级协同空间支持素材、交付、修改意见、沟通留痕。  
3. 验收后收益入账，学员端可查看收益；榜单支持多维排行。  
4. 后台支持项目、学员、审核、财务、榜单和数据全链路运营。  

### 2.2 本期不解决
1. 交付规范类字段暂不做强校验，仅提示不阻断上传。  
2. 实际提现流程仍线下处理，系统仅展示“可结算 / 可打款余额”。  
3. 更复杂的自动化质检、自动化结算、全量风控策略暂不纳入一期。  

## 3. 范围 / 非范围

### 3.1 本次范围
- 项目大厅 / 榜单
- 项目详情
- 接单
- 项目协同空间
- 素材下载与成品上传
- 账户与收益看板
- 后台管理面板

### 3.2 非本次范围
- 强校验型交付质检
- 系统内在线提现
- 更复杂的自动化推荐、风控与高级运营规则
- 完整归档保留期自动化能力

## 4. 角色定义

| 角色 | 角色说明 | 可见范围 | 可操作范围 | 特殊权限 |
|---|---|---|---|---|
| 学员 / 接单方 | 接单执行者 | 项目大厅、项目详情、协同空间、个人收益 | 浏览项目、接单、下载素材、上传交付、沟通 | 仅可见本人相关订单与收益 |
| 甲方 | 验收与反馈方 | 项目与交付结果、协同空间指定内容 | 发布修改意见、执行验收 | 验收节点开放 |
| 运营 | 平台运营者 | 项目台账、报名审核队列、订单状态、榜单配置 | 发布/下架项目、指定接单、审核报名、跟进进度、导出报表 | 统筹与项目负责人 |
| 审核 | 内容审核者 | 审核队列、交付版本、AI 预审结果 | 人工审核通过 / 退回 | 最终审核结论 |
| 财务 | 对账与结算人员 | 账本、收益、扣款、结算 | 导出对账、录入扣款、标记已结算 | 财务数据全量可见 |
| 管理员 | 系统管理员 | 全局 | 权限配置、系统配置、审计日志 | 超级权限 |

## 5. 核心业务对象

### 5.1 项目（Project）
用于承载项目需求、交付规则、接单配置、素材包与项目台账信息。

核心属性包括：
- 基础信息：项目名称、固定标签、内容类别、内容分类、难度、参与团队
- 成本预算：预算、支出、项目备注、甲方反馈
- 交付排期：交付周期、截止时间、集数、制作天数、制作开始时间
- 人员组织：参与成员、统筹人员
- 接单配置：接单模式、名额、报名审核策略、指定候选人
- 需求与素材：需求文本、需求附件、素材说明、素材包、交付要求模板字段

### 5.2 订单（Order）
接单成功后生成的执行单元，是审核、验收、收益入账、对账结算的核心对象。

### 5.3 报名申请（Application）
仅在报名模式下存在。报名成功不等于接单成功；运营通过后才转为订单。

### 5.4 协同空间（Workspace）
按订单创建，用于承接素材、交付版本、修改意见、沟通、审核与验收记录。

### 5.5 账本（Ledger）
以“已验收”作为入账节点，用于支撑收益看板、财务对账、结算标记与榜单收益口径。

## 6. 对象关系说明
- 一个项目可对应多个订单。
- 一个项目可对应多个报名申请。
- 一个订单对应一个订单级协同空间。
- 一个账本记录以订单为基础生成。
- 报名申请通过后，才生成订单与协同空间。

### 6.1 生命周期联动
- 项目下架后，前台不可见，但历史订单仍保留。
- 项目关闭 / 归档后，不再允许新增接单，但已存在订单需继续完成后续流程。
- 报名未通过 / 已取消，不得生成订单和协同空间。
- 订单验收后，收益进入账本，并同步影响收益看板与榜单。
- 不允许出现“无项目归属的订单”或“无订单归属的协同空间”。

## 7. 总体流程
### 7.1 主流程
1. 运营创建并发布项目  
2. 学员在项目大厅浏览项目  
3. 学员按抢单或报名模式参与项目  
4. 系统生成订单并自动创建协同空间  
5. 学员下载素材、上传交付版本  
6. 审核执行通过或退回  
7. 甲方执行验收  
8. 验收完成后收益入账  
9. 财务执行对账、扣款调整、结算标记  
10. 榜单与收益看板同步更新  

### 7.2 报名模式补充流程
1. 学员提交报名  
2. 系统生成报名记录，不生成订单  
3. 运营审核通过 / 驳回  
4. 通过后生成订单与协同空间  
5. 驳回后记录驳回原因并通知学员  

## 8. 状态机
### 8.1 订单状态
| 状态 | 含义 | 允许操作 | 下一状态 |
|---|---|---|---|
| ToBeTaken | 待接单 | 接单 / 报名 | InProgress |
| InProgress | 已接单进行中 | 上传交付 | Submitted |
| Submitted | 已提交待审核 | 审核通过 / 退回 | Approved / NeedRevision |
| NeedRevision | 退回修改 | 重新提交 | Submitted |
| Approved | 审核通过待验收 | 验收 | Accepted |
| Accepted | 已验收 | 入账、等待结算 | Closed / 已结算 |
| Closed | 已关闭 | 仅查看 | 无 |

### 8.2 项目状态
| 状态 | 含义 |
|---|---|
| Draft | 草稿 |
| Published | 已发布 |
| ClosedForTaking | 已满 / 已截止 |
| Closed | 已关闭 |
| Archived | 已归档 |

### 8.3 学员状态
| 状态 | 含义 |
|---|---|
| PendingApproval | 待入驻审核 |
| Training | 在训 |
| Active | 可接单 |
| Paused | 暂停 |
| Frozen | 冻结 |
| Exited | 退出 |

### 8.4 报名记录状态
| 状态 | 含义 |
|---|---|
| Pending | 待审核 |
| Approved | 已通过 |
| Rejected | 已驳回 |
| Cancelled | 已取消 |

### 8.5 逆向流转定义
- 报名在待审核阶段可撤回，撤回后状态变为 `Cancelled`。
- 撤回报名后是否允许再次报名，属于待确认策略。
- 一期未定义已验收订单的财务冲抵 / 冲正流程，后续若涉及扣款或结算回退，建议通过新账本记录处理，而不是直接覆盖原记录。

### 8.6 触达规则
关键状态变化需定义触达：
- 报名提交：通知运营进入待审核队列，同时通知学员“等待审核”
- 报名驳回：通知学员并展示原因
- 审核退回：通知学员查看修改点
- 审核通过：通知甲方可验收
- 验收完成：通知学员收益已入账
- 批量任务完成 / 失败：通知发起人查看结果

## 9. 功能模块设计
### 9.1 项目大厅 / 榜单
支持按分类展示项目，并支持分类、标签、截止时间区间、状态等筛选；支持推荐、单价、截止时间排序。榜单支持总接单量、总收益、近期活跃度三种维度。

### 9.2 项目详情
展示项目需求、交付排期、交付要求、接单规则、素材说明等核心信息，并承接接单入口。

### 9.3 接单
支持抢单模式和报名模式，确保幂等。抢单模式下接单成功即生成订单。报名模式下先生成报名记录，运营通过后生成订单。

### 9.4 协同空间
按订单生成，承接概览、素材、交付、修改意见、沟通、审核与验收。

### 9.5 素材下载与成品上传
仅空间成员可访问素材与成品文件；上传后生成新版本并填写变更说明。

### 9.6 账户与收益看板
展示累计收益、可提现余额、已完成订单数，并提供收益明细账本。

### 9.7 后台管理面板
包含项目管理、学员管理、内容审核、财务对账、榜单与数据看板。

## 10. 页面 / 后台详细设计
### 10.1 项目大厅
- 分类 Tab：短剧 / AI 漫剧 / 短视频 / 综艺
- 筛选项：分类、题材标签、截止时间区间、状态
- 排序项：推荐、单价、截止时间
- 项目卡片：项目名称、固定标签、内容类别、难度、交付周期、集数、参与团队、接单模式、名额、状态

### 10.2 协同空间
- 概览：订单状态、截止时间、成员、里程碑
- 素材：素材列表与下载
- 交付：版本列表
- 修改意见：按版本关联
- 沟通：站内图文沟通
- 审核与验收：审核结论与验收记录

### 10.3 后台项目管理
- 项目台账字段：项目 ID、名称、分类、单价、集数、截止时间、名额/已接单数、状态、负责人、订单统计
- 操作：新建、编辑、发布、下架、关闭、归档、导出

### 10.4 查询性能约束
- 项目台账、审核队列、账本列表建议默认带时间范围
- 对账导出建议支持异步导出
- 对于订单统计、榜单看板、明细钻取，需避免无限制全量查询

### 10.5 组件化规范
- 搜索区、列表区、弹窗、批量操作区应尽量复用后台统一组件
- 审核队列、项目台账、人才库等列表建议沿用统一表格与筛选组件
- 若出现新的版本管理组件或协同评论组件，建议评估是否沉淀为公共组件

## 11. 字段与交互规则
### 11.1 关键字段示例
| 字段 | 含义 | 类型 | 必填 | 规则 |
|---|---|---|---|---|
| project_name | 项目名称 | string | 是 | ≤50字 |
| content_genre | 内容类别 | enum | 是 | 霸总/穿越/男频/女频 |
| category | 内容分类 | enum | 是 | 短剧/AI漫剧/短视频/综艺 |
| difficulty | 难度 | enum | 是 | 精品/工厂 |
| budget_amount | 预算 | number | 是 | ≥0 |
| due_time | 截止时间 | datetime | 是 | 未来时间 |
| take_mode | 接单模式 | enum | 是 | 抢单/报名 |
| quota_total | 名额 | int | 是 | ≥1 |
| requirement_text | 项目需求文本 | text | 是 | ≤5000字 |

### 11.2 交互规则
- 接单按钮受学员状态、并发上限、项目状态、名额约束共同控制
- 报名按钮需要保证幂等
- 重复点击接单 / 报名不得生成重复记录
- 上传交付后自动生成新版本并进入 Submitted
- 审核退回时，退回原因和修改点必填

### 11.3 敏感字段
本期原始 PRD 未显式列出手机号、身份证号等 PII 字段，但涉及财务账本、结算状态与收益金额。后续若扩展到实名信息、手机号、银行卡号，应补充：列表页与详情页脱敏规则、导出权限、明文查看权限、审计日志要求。

## 12. 业务规则与配置规则
### 12.1 核心规则
- 入账节点：验收后入账
- 总收益口径：已验收订单金额 - 扣款调整
- 榜单收益允许公开展示
- 并发上限是系统配置项，可对个人 / 项目覆盖
- 接单后自动创建订单级协同空间
- 报名阶段不占用并发产能

### 12.2 配置类能力
- 报名审核策略：自动通过 / 运营审核
- 榜单维度：总接单量 / 总收益 / 近期活跃度
- 榜单周期：总榜 / 月榜
- 刷新策略：每日 / 每小时可配置
- 活跃度权重可配置

### 12.3 配置生效验证机制
建议补充以下平台治理能力：
- 是否区分草稿版本与线上生效版本
- 是否支持查看规则差异
- 配置保存后是否需发布
- 缓存刷新或榜单重算的时效
- 若配置未生效，如何排查

## 13. 权限规则
### 13.1 页面权限
- 项目空间仅项目成员可见
- 财务账本仅财务 / 管理员可见
- 审核页面仅审核角色可进入
- 系统配置与审计日志仅管理员可见

### 13.2 按钮权限
- 项目上下架：运营 / 管理员
- 报名通过 / 驳回：运营
- 审核通过 / 退回：审核
- 扣款调整 / 已结算标记：财务
- 权限配置：管理员

### 13.3 审计要求
以下动作必须审计留痕：
- 接单
- 上传
- 审核退回
- 审核通过
- 验收
- 扣款
- 结算标记
- 项目上下架
- 管理员隐藏 / 作废版本

## 14. 技术约束与非功能需求
- 空间与文件需强鉴权
- 敏感数据需分权
- 核心链路可用性目标 ≥99.5%
- 大文件上传建议分片 / 断点续传
- 关键动作应可检索、可导出
- 后续建议补：历史数据保留期、归档 TTL、异步重试与断点恢复策略

## 15. 数据口径
### 15.1 成功指标
- 接单转化率 = 接单成功人数 / 项目详情 UV
- 准时交付率 = 提交时间 ≤ 截止时间的订单数 / 已接单订单数
- 验收闭环率 = 已验收订单数 / 已提交订单数
- 平均交付周期 = 首次提交时间 - 接单时间
- 平均审核周期 = 审核结论时间 - 提交时间
- 平均修改轮次 = 退回次数 + 1
- 总收益 = Σ（已验收订单金额）- 扣款调整

### 15.2 口径统一要求
- 榜单、收益看板、账本、财务报表都应围绕统一账本口径
- 同一订单不得重复计入收益与榜单
- 明细应支持钻取到项目 / 订单层

## 16. 日志 / 审计 / 埋点
建议最少覆盖以下日志：
- 接单日志
- 报名审核日志
- 上传版本日志
- 审核日志
- 验收日志
- 财务扣款与结算日志
- 项目上下架日志
- 榜单配置日志
- 审计导出日志

## 17. 验收标准
### 17.1 功能验收
1. 项目能按分类展示，状态准确
2. 榜单支持多维度切换，且与账本口径一致
3. 一键接单成功且幂等
4. 报名模式可查看待审核/已通过/已驳回/已取消状态
5. 接单成功后自动创建订单级工作空间
6. 支持至少 3 轮提交 - 退回 - 再提交
7. 验收后收益入账，学员端看板可见
8. 后台可完成项目管理、学员管理、内容审核、财务对账、榜单看板

### 17.2 异常与边界验收
1. 重复点击接单不生成重复订单
2. 重复点击报名不生成重复申请
3. 驳回原因必填
4. 未通过 / 已取消的报名不生成订单与协同空间
5. 非成员无法访问素材与成品
6. 验收后版本锁定

## 18. MVP 演进建议
### 18.1 一期最小闭环
优先保证：
- 发布项目
- 接单 / 报名
- 订单级协同空间
- 审核退回 / 通过
- 验收入账
- 财务对账与结算标记

### 18.2 可延后到 Phase 2 的能力
- 更强的交付规范自动校验
- 更复杂的榜单重算与手动校验能力
- 自动推荐接单
- 配置版本管理与预校验
- 自动归档 / TTL
- 更完整的断点恢复与任务编排
- 更细的敏感信息字段治理

## 19. 待确认清单
- [待确认] 报名撤回后是否允许重新报名  
- [待确认] 活跃度榜权重是否固定为当前默认值，还是需要完整后台可调  
- [待确认] 榜单刷新策略是否需要支持手动重算  
- [待确认] 素材与成品保留期是否需要在一期显式配置  
- [实现口径待明确] 财务扣款调整是否需要支持附件上传与审批链  
- [实现口径待明确] 项目关闭 / 归档后，未完成订单的后续操作边界
