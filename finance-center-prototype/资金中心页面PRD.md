# 需求背景与范围

## 1. 现状

天合平台存在需求发布、订单支付、预算冻结、验收结算、退款、提现、积分兑换和发票申请等资金动作。当前钱包模块信息层级不够清晰，主要问题包括：

- 钱包首页与费用成本类页面相比，缺少清晰的账户总览、账单追溯和发票入口。
- 用户同时可能具备需求方和承接方身份，但页面容易让用户误以为存在两套钱包。
- 平台同时存在个人账户和团队账户，用户加入多个团队后，需要明确当前资金口径。
- 交易流水、资金流水、账单明细概念容易混用，用户侧应优先展示可理解的账单明细。
- 充值、开票等动作应该在当前账户上下文内完成，避免跳转后丢失账户口径。

## 2. 本次目标

1. 建立「统一账户 + 个人/团队账户口径」的资金中心页面结构。
2. 明确资金总览、账户、交易流水、资金流水、账单明细、充值中心、积分中心、发票中心的页面边界。
3. 让用户进入资金中心后能快速判断：当前看的是个人账户还是团队账户。
4. 让团队账户用户能查看团队预算、支出、冻结、开票和充值。
5. 让个人账户用户能查看个人收益、提现、充值、积分和最近账单。
6. 将充值、申请开票等高频动作设计为弹窗，保留当前页面上下文。
7. 为研发与测试提供页面字段、交互规则、异常处理和验收标准。

## 3. 非目标（本迭代不做）

- 不重构底层资金账户、交易流水、资金流水数据库模型。
- 不实现后台财务审核页、后台对账页、资金流水后台管理页。
- 不实现上传转账凭证能力。
- 不实现成本优化、费用预测、外部云厂商账单对接能力。
- 不实现发票自动审核、自动开具、税控系统直连。
- 不实现支付渠道真实收银台，仅在原型层展示支付渠道选择。
- 不实现优惠券、礼品卡等非当前业务已确认能力。

## 6. 一期已确认产品口径

| 决策项 | 一期口径 |
|---|---|
| 个人账户是否允许申请发票 | 不允许。发票中心一期仅按团队账户 / 企业主体处理，个人账户下隐藏申请入口或展示无可开票数据。 |
| 团队账户充值是否保留对公转账 | 不保留新增入口。发起充值仅支持在线充值，支付渠道为支付宝、微信。 |
| 充值中心默认时间范围 | 默认最近 30 天。 |
| 账单明细默认查询时间范围 | 默认最近 30 天。 |
| 开票税率枚举与默认值 | 枚举为 6% / 3% / 1%，默认 6%，后续以财务配置为准。 |
| 团队成员是否可发起团队充值 | 普通团队成员不可发起；仅团队负责人 / 财务负责人可发起团队充值。 |
| 发票申请驳回后是否允许原单修改再提交 | 允许原单修改后重新提交，状态从审核驳回回到待审核，并保留驳回和修改记录。 |
| 下载凭证类型与生成规则 | 一期支持账单明细凭证 PDF，包含账单编号、业务单号、账户、金额、时间、状态。 |
| 充值成功后余额更新 | 支付成功回调后实时刷新余额、充值记录和账单明细；回调延迟时展示「支付成功，入账处理中」。 |
| 历史充值记录中的银行转账 / 企业总池数据 | 一期继续展示历史数据，但不提供新增入口；历史支付方式不出现在新充值下拉选项中。 |

# 账户模型与核心业务对象

## 1. 统一账户模型

平台用户可能同时具备：

- 需求方身份
- 承接方身份
- 团队成员身份
- 团队负责人或财务身份

系统采用：

```text
统一钱包账户
+ 统一资金体系
+ 统一账单体系
```

不拆分：

```text
需求方钱包
承接方钱包
```

需求方充值、需求方消费、需求方退款、承接方收入、承接方结算、承接方提现共存于同一资金体系，通过 `business_role` 区分。

## 2. 账户口径

| 账户口径 | 定义 | 典型资金动作 |
|---|---|---|
| 个人账户 | 用户个人收益和个人余额账户 | 承接方收入、提现、个人充值、积分兑换 |
| 团队账户 | 团队预算账户 | 团队充值、需求发布、订单支付、预算冻结、退款回补、开票 |

### 顶部账户切换规则

| 用户操作 | 系统反馈 |
|---|---|
| 选择团队账户 | 显示当前团队下拉；页面按团队账户渲染 |
| 选择个人账户 | 隐藏当前团队下拉；页面按个人账户渲染 |
| 个人账户访问资金总览 | 自动跳转到账户页 |
| 切换团队 | 当前页面数据按新团队刷新 |

## 3. 核心业务对象

| 对象 | 说明 |
|---|---|
| 钱包账户 | 资金归属账户，可为个人账户或团队账户 |
| 资金科目 | 可用余额、冻结余额、待结算收入、可提现金额、提现中金额、积分余额等 |
| 充值单 | 用户发起充值后生成的业务单据 |
| 交易流水 | 平台交易事件记录，如充值、冻结、结算、退款、提现等 |
| 资金流水 | 账户资金科目余额变动记录，保留变动前余额、变动金额、变动后余额 |
| 账单明细 | 用户可理解的资金变动展示记录 |
| 发票抬头 | 发票主体资料 |
| 可开票账单 | 满足开票条件的账单明细 |
| 开票申请 | 用户提交的发票申请单 |
| 电子发票 | 已开具并可下载的发票文件记录 |

## 4. 三类流水关系

资金中心页面需要区分：

1. 交易流水：记录一笔交易事件。
2. 资金流水：记录某个账户某个资金科目的余额变动。
3. 账单明细：面向用户展示的钱包账单记录。

关系：

```text
一条交易流水 -> 多条资金流水
多条资金流水 -> 可聚合为一条账单明细
账单明细 -> 面向用户展示，并可追溯交易流水和资金流水
```

本 PRD 覆盖资金中心内的交易流水、资金流水和账单明细页面，但不覆盖后台财务审核、后台对账和底层账务系统实现。

## 5. 现金退款业务口径

本 PRD 中的「退款」默认指资金账户内的现金退款，不包含积分兑换后的退现金。

| 场景 | 处理口径 | 页面展示 |
|---|---|---|
| 订单取消退款 | 生成退款单 RF，退款金额回补到原付款账户可用余额 | 账单明细展示「退款到账」，交易流水展示「退款」，资金流水展示可用余额入账 |
| 订单争议部分退款 | 按实际退款金额回补现金余额 | 账单明细展示部分退款金额，状态可为处理中 / 成功 |
| 重复支付 / 异常支付退款 | 退款至原支付账户或当前钱包余额，[实现口径待支付侧确认] | 充值记录或账单详情展示退款状态 |
| 已冻结未结算金额退款 | 先解冻，再按退款规则回补可用余额 | 资金流水体现冻结减少和可用余额增加 |
| 积分兑换后退现金 | 不支持。积分兑换成功后不可申请现金退款 | 积分中心展示积分消费/退回，不展示现金退款入口 |
| AIGC 消费失败 | 仅退回积分，不退现金 | 积分中心展示「消费失败退回」 |

现金退款需要保留原交易流水号、原资金流水号和退款单号，便于客服、财务和用户核对。

### 5.1 退款入口规则

| 入口 | 展示规则 |
|---|---|
| 原订单 / 订单资金详情 | 作为现金退款主入口。未结算订单展示「申请退款」，已结算或争议订单展示「联系客服退款」和「查看退款进度」 |
| 账单详情 | 作为辅助入口。冻结中订单账单展示「申请退款」，退款账单展示「查看退款进度」，充值异常展示「联系客服退款」，不可退款账单展示「不可退款」 |
| 充值详情 | 仅在重复支付、异常支付、充值未到账等场景展示「联系客服退款」 |
| 交易流水 / 资金流水 | 不提供退款操作，只用于查询和追溯 |
| 积分中心 | 不提供现金退款入口；积分兑换成功后不支持退现金 |

## 3. 数据权限

- 个人账户数据只能展示当前登录用户相关记录。
- 团队账户数据只展示当前用户有权限访问的团队。
- 用户加入多个团队时，顶部团队下拉仅展示其可访问团队。
- 个人账户下不展示「当前团队」选择器，避免误解为个人账户挂靠某个团队。

# 积分中心 · 页面交互说明

> 对应原型：`finance-center-prototype/points.html`。积分中心用于展示积分余额、积分兑换、充值并兑换入口、积分明细和 AIGC token 消费记录。积分归属于统一账户，但不等同现金余额。

## 一、页面入口

| 入口 | 说明 |
|---|---|
| 左侧菜单「积分中心」 | 查看当前账户口径下积分资产 |
| 账户页「积分余额」 | 从账户摘要进入积分详情 |
| 积分中心「充值并兑换」 | 在当前弹窗内完成充值和兑换积分 |
| AIGC 平台消费记录 | 从 token 消费追溯积分扣减 |

## 二、页面结构

1. 页面说明：说明积分是统一账户下的虚拟权益资产。
2. 指标区：积分余额、本月获得积分、本月消耗积分、预计可用 token、处理中积分。
3. 兑换区：余额兑换积分、充值并兑换积分入口。
4. 查询区：积分动作、关联业务单号、变动方向、账户类型、状态、发生时间范围。
5. 积分明细：展示积分获得、消耗、退回等记录。
6. AIGC token 消费记录：展示 token 消耗到具体 AIGC 任务的记录。
7. 兑换弹窗：填写兑换金额；余额不足时引导补差充值并兑换。
8. 充值并兑换弹窗：在一个弹窗内完成充值金额、支付渠道和兑换确认。

## 三、核心规则

| 规则 | 说明 |
|---|---|
| 余额兑换积分 | 用户可直接使用当前账户余额兑换积分 |
| 余额不足 | 在兑换弹窗内展示差额，并提供「补差充值并兑换」按钮；补差页只充值差额，支付后合并当前余额完成原目标兑换 |
| 充值并兑换积分 | 用户无需跳转充值中心，可在积分中心弹窗内完成支付和兑换；从顶部入口进入时充值多少兑换多少，从补差入口进入时按差额补足 |
| AIGC token 消费 | 用户在 AIGC 平台消费 token 后扣减积分 |
| 积分兑换退现金 | 不支持。积分兑换成功后不可申请现金退款 |
| 消费失败退回 | 仅回补积分，不退现金 |
| 流水联动 | 积分兑换和 token 消费均需关联交易流水、资金流水和账单明细 |
| 发票口径 | 积分充值/兑换是否开票暂不纳入一期，后续由财务税务确认 |

| 积分动作 | 余额兑换积分 / AIGC token 消费 / 消费失败退回 / 承接奖励积分 |
| 变动方向 | 获得 / 消耗 / 退回 |
| 积分变动 | 本次积分增减数量 |
| 关联金额 | 兑换积分时对应现金金额 |
| 关联 token | AIGC 消费时对应 token 数量 |
| 关联业务单号 | PT / AI / RC 等业务单号 |
| 状态 | 成功 / 处理中 / 失败 / 已退回 |
