# 资金中心逐页深化计划

## 目标
把已经拆成多页面的原型，从“结构正确”推进到“可以评审”的状态。

## 原则
- 一次只深打一页或一组强关联页面。
- 每轮都补齐：信息层级、图表、筛选、状态、空态、异常态、按钮动作。
- 先打通前台主链路，再打订单资金详情，最后打后台治理页。

## 页面批次

### 第一批：前台主入口
1. `overview.html`
2. `accounts.html`

目标：
- 把首页和账户页打到可以作为评审首页的质量。
- 明确团队预算首页的视觉主叙事。

需要补的重点：
- 更强的首屏层级和更真实的图表风格
- 团队切换后的状态差异
- 指标卡点击后的下钻入口
- 空态、异常态、预算不足态

### 第二批：事实查询链路
1. `transactions.html`
2. `order-finance-detail.html`

目标：
- 打通“首页看异常 -> 查流水 -> 回订单看资金轨迹”的主链路。

需要补的重点：
- 更完整的筛选器
- 流水详情抽屉 / 侧栏
- 订单资金时间轴细化
- 凭证、服务费、退款、开票字段补全

### 第三批：月度与税务链路
1. `bills.html`
2. `invoices.html`

目标：
- 让财务、团队负责人能清楚区分账单和发票。

需要补的重点：
- 月度趋势图丰富
- 账单详情层
- 发票状态流转
- 抬头管理与电子票查看

### 第四批：资金动作页
1. `recharge.html`
2. `withdraw.html`
3. `security.html`

目标：
- 把“补钱、提钱、支付安全”变成更真实的业务表单和处理页。

需要补的重点：
- 表单校验态
- 多种充值方式展开
- 提现失败 / 驳回 / 到账中的状态页
- 银行卡管理和限额设置

### 第五批：后台治理页
1. `admin-overview.html`
2. `admin-budget.html`
3. `admin-audit.html`
4. `admin-settlement.html`

目标：
- 形成一套可讨论的后台财务中心结构。

需要补的重点：
- 企业财务指标层
- 审核处理工作台
- 预算划拨操作面板
- 发票处理与结算处理流程

## 每页深化验收标准
- 页面首屏能说明“这是干什么的”
- 至少有一个高质量图表或高信息密度列表
- 有主状态、空态、异常态
- 有核心按钮和后续跳转关系
- 金额、状态、口径表达一致

## 建议顺序
1. 先打 `overview.html`
2. 再打 `transactions.html`
3. 然后打 `order-finance-detail.html`
4. 再处理 `bills.html` 和 `invoices.html`
5. 最后补后台页
